Vector store RAG

Retrieval Augmented Generation(RAG)은 LLM(대형 언어 모델)을 사용자의 데이터로 학습시키고, 해당 데이터에 대해 질의할 수 있는 방식입니다.

RAG는 벡터 스토어, 즉 입력된 데이터의 임베딩을 저장하는 벡터 데이터베이스를 기반으로 동작합니다.

이로 인해 벡터 검색이 가능해지며, 이는 더욱 강력하고 문맥을 이해하는 검색을 제공합니다.

이 스타터 플로우에서는 Astra DB를 벡터 데이터베이스로 선택했지만, Langflow의 다양한 벡터 데이터베이스 옵션을 활용해도 무방합니다.

사전 요구사항

  • 실행 중인 Langflow 인스턴스
  • OpenAI API 키
  • Astra DB 벡터 데이터베이스 (다음 조건 포함)
    • 데이터베이스에 읽기 및 쓰기 권한이 있는 Astra DB 애플리케이션 토큰
    • Astra에서 생성된 컬렉션 또는 Astra DB 컴포넌트에서 새로 생성된 컬렉션

Langflow를 열고 새 프로젝트를 시작하세요.

  1. Langflow 대시보드에서 New Flow를 클릭합니다.
  1. Vector Store RAG를 선택합니다.
  1. Vector Store RAG 플로우가 생성됩니다.

벡터 RAG 플로우 구축

벡터 스토어 RAG 플로우는 데이터 입력(ingestion)과 질의(query)를 위한 두 개의 별도 플로우로 구성됩니다.

Load Data Flow(화면 하단)는 문맥적 유사성 검색을 위한 검색 가능한 인덱스를 생성합니다. 이 플로우는 로컬 파일의 데이터를 벡터 스토어에 저장합니다. 로컬 파일에서 데이터를 불러와 청크 단위로 분할하고, Astra DB에 인덱싱하며, OpenAI 임베딩 모델을 사용해 각 청크의 임베딩을 계산합니다.

  • Chat Input은 Playground에서 사용자의 입력을 받습니다.
  • OpenAI Embeddings는 사용자 질의를 벡터 형태로 변환합니다.
  • Astra DB는 질의 벡터를 사용해 유사도 검색을 수행합니다.
  • Parser는 검색된 청크를 처리합니다.
  • Prompt는 사용자 질의와 관련된 문맥을 결합합니다.
  • OpenAI는 해당 프롬프트를 이용해 답변을 생성합니다.
  • Chat Output은 생성된 답변을 Playground로 반환합니다.
  1. OpenAI 모델 컴포넌트 설정
    1. OpenAI 컴포넌트에 전역 변수를 만들려면 OpenAI API Key 필드에서 지구본(Globe) 버튼을 클릭한 후 Add New Variable을 선택하세요.
    1. Variable Name 필드에 openai_api_key를 입력합니다.
    1. Value 필드에 본인의 OpenAI API Key(sk-...)를 붙여넣습니다.
    1. Save Variable을 클릭하여 저장합니다.
  1. Astra DB 컴포넌트 설정
    1. Astra DB Application Token 필드에 본인의 Astra DB 애플리케이션 토큰을 입력하세요. 컴포넌트가 데이터베이스에 연결되며, 기존 데이터베이스와 컬렉션이 메뉴에 표시됩니다.
    1. Database를 선택합니다. 컬렉션이 없다면 New database를 선택하세요. Name, Cloud provider, Region 필드를 입력한 뒤 Create를 클릭합니다. 데이터베이스 생성에는 몇 분이 소요될 수 있습니다.
    1. Collection을 선택합니다. 컬렉션은 Astra DB 배포 환경에서 벡터 데이터를 저장하기 위해 생성됩니다.

  1. 컬렉션이 없다면 컴포넌트 내에서 새 컬렉션을 만드세요.
    1. New collection을 선택하세요.
    1. Name, Embedding generation method, Embedding model, Dimensions 필드를 입력한 후 Create를 클릭하세요.

      Embedding generation method와 Embedding model의 선택은 임베딩을 Astra의 vectorize 서비스를 통해 제공되는 모델로 생성할지, Langflow의 컴포넌트에서 직접 생성할지에 따라 달라집니다.

      1. Astra의 vectorize 서비스를 통해 임베딩을 생성하려면, Embedding generation method 드롭다운에서 모델을 선택하고, Embedding model 드롭다운에서도 모델을 선택하세요.
      1. Langflow의 컴포넌트에서 임베딩을 생성하려면, Embedding generation method와 Embedding model 필드 모두에서 Bring your own을 선택하세요. 이 스타터 프로젝트에서는 OpenAI Embeddings 컴포넌트를 Astra DB 컴포넌트에 연결해 사용하는 옵션이 해당됩니다.
      1. Dimensions 값은 컬렉션의 임베딩 차원과 일치해야 합니다. Astra의 vectorize 서비스를 통해 임베딩을 생성하는 경우에는 이 필드가 필수가 아닙니다. 이 값은 Astra DB 배포 환경의 Collection에서 확인할 수 있습니다.

      자세한 내용은 DataStax Astra DB Serverless 문서를 참고하세요.

    Langflow의 Global Variables 기능을 사용했다면, RAG 애플리케이션 플로우 컴포넌트는 이미 필요한 자격 증명으로 설정되어 있습니다.

    Vector Store RAG 플로우 실행

    1. Playground를 클릭하세요. 여기서 생성한 데이터베이스의 컨텍스트를 활용하는 AI와 대화할 수 있습니다.
    1. 메시지를 입력하고 Enter를 누르세요. (예: "What topics do you know about?"와 같이 입력해보세요.)
    1. 봇이 임베딩한 데이터의 요약 정보를 답변으로 제공합니다.
