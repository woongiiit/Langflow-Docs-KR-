Vector stores

벡터 스토어 (Vector Store) 컴포넌트

: Components > Vector Stores

Langflow의 Vector Store 컴포넌트는 벡터 데이터를 저장하고 검색하는 기능을 통해 챗봇이나 RAG(Retrieval-Augmented Generation) 같은 AI 작업을 지원합니다.

Vector Store 컴포넌트는 기존의 벡터 데이터베이스와 연결하거나, Langflow 내에서 임시로 벡터 저장소를 생성할 수 있습니다. 이렇게 저장된 벡터 데이터는 나중에 검색과 유사도 비교 작업에 사용됩니다.

메모리 컴포넌트와는 다릅니다. 메모리 컴포넌트는 주로 채팅 메시지를 저장하고 불러오기 위한 용도로 설계되어 있으며, 내부 메모리 또는 외부 DB와 연결됩니다. 자세한 내용은 메모리 관리 옵션을 참고하세요.

플로우에서 Vector Store 컴포넌트 사용하기

다음 예시는 Chroma DB 컴포넌트를 사용하는 경우입니다. 사용 중인 Vector Store 컴포넌트에 따라 매개변수나 인증 방식은 달라질 수 있으나, 전체 문서 업로드 및 임베딩(embedding) 처리 방식은 유사합니다.

  1. 로컬 파일에서 문서를 불러온 후, 텍스트를 청크로 나눕니다.
  1. 연결된 임베딩 모델(model) 컴포넌트가 텍스트를 벡터로 변환합니다.
  1. 변환된 벡터는 Vector Store에 저장됩니다.

이렇게 저장된 벡터 데이터는 RAG와 같은 워크로드에서 검색에 사용될 수 있습니다.

사용자의 입력값 역시 임베딩되어, 사전에 저장된 벡터들과 유사도를 비교합니다.

  1. Vector Store 컴포넌트는 유사도 검색 결과를 Data 객체로 출력합니다.
  1. 이 Data 객체는 텍스트로 변환된 뒤, Prompt 컴포넌트의 {context} 변수에 삽입됩니다.
  1. 해당 context는 OpenAI 모델 컴포넌트가 응답을 생성할 때 참고하게 됩니다.

임베딩 생성 방식

Astra DB Vector Store는 두 가지 방식으로 임베딩을 생성할 수 있습니다.

  1. 임베딩 모델 연결

    Langflow의 Embedding 컴포넌트를 연결해 직접 임베딩을 생성합니다.

  1. Astra Vectorize 사용

    Astra DB 자체의 벡터화 서비스(Astra Vectorize)를 사용할 수도 있습니다. 컬렉션을 생성할 때, NVIDIA의 NV-Embed-QA와 같은 모델을 선택할 수 있습니다.

임베딩 모델은 컬렉션 생성 시 지정되며, 생성 이후에는 변경할 수 없습니다.

관련 예시는 Vector Store RAG 스타터 프로젝트를 참고하세요.

자세한 정보는 Astra DB 임베딩 생성 문서를 참고하세요.

Hybrid Search (혼합 검색)

Astra DB는 기본적으로 Hybrid Search를 지원합니다.

Hybrid Search는 다음 세 가지 항목으로 구성됩니다:

  • Search Query: 벡터 기반 유사도 검색을 위한 질의어
  • Lexical Terms: 쉼표로 구분된 키워드 목록 (예: features, data, attributes, characteristics)
  • Reranker: 벡터 결과와 키워드 결과를 재정렬하는 모델 (nvidia/llama-3.2-nv.reranker)

Hybrid Search는 벡터 유사도 검색과 키워드 기반 검색을 모두 수행하고, 그 결과를 비교한 뒤 최적의 결과를 반환합니다.

Hybrid Search를 사용하려면 컬렉션을 생성할 때 다음 기능이 활성화되어 있어야 합니다: 벡터, 텍스트 검색(lexical), 재정렬(rerank).

AWS us-east-2 리전에서 데이터베이스를 만들면 기본적으로 활성화됩니다.

자세한 설정 방법은 DataStax 문서를 참고하세요.

Hybrid Search 사용 방법

  1. New Flow > RAG > Hybrid Search RAG를 선택하세요.
  1. OpenAI 모델 컴포넌트에 OpenAI API 키를 입력합니다.
  1. Astra DB Vector Store 컴포넌트에 Astra DB Application Token을 입력합니다.
  1. Database 필드에서 사용할 데이터베이스를 선택합니다.
  1. Collection 필드에서 Hybrid 기능이 활성화된 컬렉션을 선택하거나 새로 생성합니다.
  1. Playground에서 다음과 같이 질문을 입력합니다:

    What are the features of my data?

이때 사용자의 입력은 두 컴포넌트로 전송됩니다:

  • OpenAI 모델: 사용자의 질문을 기반으로 키워드와 임베딩용 질문을 생성합니다.
  • Astra DB: 벡터 기반 및 키워드 기반 검색을 수행합니다.

Chroma DB

Chroma DB 컴포넌트는 간단한 실험과 벡터 저장을 위한 **임시 벡터 데이터베이스(ephemeral vector database)**를 생성합니다.

Langflow 내에서 빠르게 벡터 기반 검색을 테스트하고 임베딩 데이터를 저장할 수 있는 가장 기본적인 Vector Store입니다.

플로우에서 사용하려면, Data 또는 DataFrame을 출력하는 컴포넌트에 연결하면 됩니다.

아래는 예시 흐름입니다:

  1. URL 컴포넌트에서 텍스트 데이터를 불러온 후, 텍스트를 분할합니다.
  1. OpenAI Embeddings 컴포넌트를 통해 텍스트를 임베딩합니다.

    (Chroma DB는 자체 임베딩도 가능하지만, 외부 모델 연결을 추천합니다.)

  1. Chroma DB 컴포넌트의 Collection 필드에 컬렉션 이름을 입력합니다.
  1. 영구 저장을 원할 경우, Persist 필드에 디렉토리 경로를 입력합니다.

    예: ./chroma-db → Langflow 실행 디렉토리 기준으로 chroma.sqlite3 파일이 생성됩니다.

  1. 데이터를 저장하려면 컴포넌트에서 <Icon name="Play" aria-hidden="true"/> Run component를 클릭합니다.

중복된 문서를 여러 번 저장하려면 Allow Duplicates 옵션을 활성화하세요.

비활성화하면 동일한 콘텐츠는 자동으로 중복 제거됩니다.

  1. Split Text 컴포넌트에서 <Icon name="TextSearch" aria-hidden="true"/> Inspect output을 클릭하면, 분할된 텍스트를 확인할 수 있습니다.
  1. Playground에서 쿼리를 입력하면, Chroma DB가 저장된 벡터와 입력값을 비교하여 유사도 검색을 수행합니다.

Local DB

Local DB 컴포넌트는 Langflow에서 Chroma DB를 확장한 로컬 벡터 데이터베이스입니다.

사용자 친화적인 인터페이스를 제공하며, 다음 두 가지 모드를 지원합니다:

  • Ingest 모드: 데이터를 벡터로 임베딩하고 저장
  • Retrieve 모드: 저장된 벡터 데이터를 검색

이 컴포넌트는 자동 컬렉션 관리 기능과 Langflow의 캐시 디렉터리에 저장되는 영속적 저장(persistence) 기능을 내장하고 있어,

일반적인 ChromaDB 사용보다 편리하게 데이터를 다룰 수 있습니다.

Ingest 모드는 일반 ChromaDB처럼 작동하며, 데이터는 Langflow 캐시 디렉토리에 영구 저장됩니다.

해당 디렉터리는 환경 변수 LANGFLOW_CONFIG_DIR을 통해 지정할 수 있습니다.

Retrieve 모드는 기존 Chroma 컬렉션을 질의할 수 있는 검색 기능을 제공합니다.
자세한 정보는 Chroma 공식 문서를 참고하세요.

입력 (Inputs)

이름타입설명
collection_nameString사용할 Chroma 컬렉션 이름입니다. 기본값: "langflow"
persist_directoryString벡터 저장소를 저장할 사용자 지정 디렉터리입니다. 벡터는 $DIRECTORY/vector_stores/$COLLECTION_NAME 경로에 저장됩니다. 지정하지 않으면 시스템 캐시 폴더가 사용됩니다.
existing_collectionsString기존에 생성된 컬렉션 중 하나를 선택하여 검색할 수 있습니다.
embeddingEmbeddings사용할 임베딩 함수입니다.
allow_duplicatesBooleanfalse로 설정하면 중복된 문서는 저장되지 않습니다.
search_typeString검색 방식입니다. "Similarity" 또는 "MMR" 중 선택합니다.
ingest_dataData / DataFrame저장할 데이터입니다. 입력된 데이터는 임베딩되어 검색이 가능하도록 인덱싱됩니다.
search_queryString지정한 컬렉션에서 유사한 콘텐츠를 검색할 질의어입니다.
number_of_resultsInteger반환할 검색 결과 수입니다. 기본값: 10
limitIntegerallow_duplicates=false일 때 비교할 최대 문서 수입니다.
