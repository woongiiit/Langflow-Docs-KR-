Components overview

컴포넌트 (Component) 개요

: Components > Overview

1. 컴포넌트란?

Langflow에서 컴포넌트(component)는 플로우(flow)를 구성하는 가장 기본적인 단위입니다. 쉽게 말해, 각 컴포넌트는 정해진 역할이나 기능을 담당하며, 여러 컴포넌트를 조합해 원하는 데이터 처리 흐름을 만들 수 있습니다.

예를 들어, 데이터를 불러오거나(입력), 처리하거나(가공), 결과를 출력하는(출력) 등 각각의 작업이 컴포넌트 단위로 이루어집니다.

2. 컴포넌트 추가하기


작업을 시작하려면, 먼저 원하는 컴포넌트를 화면 왼쪽의
컴포넌트 메뉴에서 끌어다 작업 공간(Workspace)에 놓으면 됩니다.

컴포넌트 종류

컴포넌트 메뉴는 용도별로 정리되어 있는데, 대표적으로 다음과 같은 종류가 있습니다:

  • 베타 컴포넌트: Langflow에서 기본적으로 제공하는 주요 기능들입니다. 아직 정식 버전이 아니므로 실제 서비스 운영에는 권장되지 않습니다.
  • 레거시 컴포넌트: 더 이상 공식 지원하지 않지만, 이전 프로젝트와의 호환을 위해 남아 있습니다. 기본적으로는 메뉴에 보이지 않지만, '컴포넌트 설정'에서 표시할 수 있습니다.
  • 번들 컴포넌트: 외부 서비스(프로바이더)와 연동하기 위한 컴포넌트로, 서비스별로 묶여 있습니다.

3. 컴포넌트 설정 및 연결

컴포넌트를 추가했다면, 이제 각각의 입력값, 출력값, 설정값(파라미터)을 원하는 대로 조정할 수 있습니다.

기본적으로 자주 쓰는 필수 옵션만 보이지만, 더 세밀한 설정이 필요하면 컴포넌트 상단 메뉴(… 또는 Show More)를 클릭해서 추가 옵션을 확인할 수 있습니다.

주요 설정 메뉴

  • Code:

    컴포넌트 내부의 Python 코드를 직접 수정할 수 있습니다.

  • Controls:

    선택 가능한 모든 파라미터(숨겨진 옵션 포함)를 한눈에 볼 수 있습니다.

  • Tool Mode:

    Agent 컴포넌트와 연동이 필요할 때 사용하는 모드입니다.

  • Show More:

    컴포넌트 삭제, 복제 등 기타 추가 기능을 제공합니다.

4. 컴포넌트 이름 및 설명 바꾸기

작업 공간에서 원하는 컴포넌트를 클릭한 후 Edit을 누르면, 이름과 설명을 자유롭게 바꿀 수 있습니다.

설명에는 마크다운(Markdown) 문법도 사용할 수 있습니다.

5. 컴포넌트 실행하기

한 개의 컴포넌트만 실행해보고 싶다면, 해당 컴포넌트를 선택한 뒤 Run component 버튼을 누르세요.

실행 결과가 정상적으로 출력되면 Last Run이라는 표시가 뜹니다.

※ 참고:

단일 컴포넌트 실행은 전체 플로우 실행과 다릅니다.

이 방식으로 실행할 경우, 직접 연결된 이전 단계(Upstream) 컴포넌트는 자동으로 실행되지 않습니다.

6. 컴포넌트 출력 및 로그 확인

실행 결과나 내부 처리 로그를 보고 싶다면, 해당 컴포넌트를 선택한 뒤 Inspect 버튼을 누르면 됩니다.

7. 컴포넌트 고정(Freeze) 기능

컴포넌트 실행 결과가 항상 동일하다면, Freeze(고정) 기능을 쓸 수 있습니다.

고정을 하면 해당 컴포넌트뿐만 아니라, 그 컴포넌트로 들어오는 상위 컴포넌트까지 모두 다시 실행되지 않고, 마지막 실행 결과가 그대로 유지됩니다.

고정하려면, 컴포넌트를 클릭 → 상단 메뉴(Show More) → Freeze를 선택하면 됩니다.

8. 컴포넌트 포트(Port)와 데이터 연결

컴포넌트에는 동그란 포트가 달려 있습니다.

이 포트에 선을 연결해 컴포넌트끼리 데이터를 주고받을 수 있습니다.

각 포트의 색상은 데이터 종류(예: 텍스트, 데이터, 임베딩 등)를 의미합니다.

마우스를 올리면 연결 가능한 데이터 유형에 대한 안내도 볼 수 있습니다.

주요 포트 색상 예시

데이터 종류포트 색상
Data빨간색
DataFrame분홍색
Embeddings초록색
LanguageModel자홍색
Memory주황색
Message/Text남색
Tool청록색
Unknown회색

9. 컴포넌트의 코드 구조와 동작 방식

모든 컴포넌트는 내부적으로 Python 코드로 동작합니다.

이 코드는 입력값, 출력값, 설정값(파라미터), 그리고 실제 동작(함수 등)을 정의합니다.

모든 컴포넌트는 Component라는 기본 클래스에서 상속받습니다. 예를 들어 RecursiveCharacterTextSplitter는 LCTextSplitterComponent를 상속한 하위 클래스입니다

이 컴포넌트는 다음과 같은 입력값을 받을 수 있습니다.

  • chunk_size(조각 크기)
  • chunk_overlap(겹치는 길이)
  • data_input(입력 데이터)
  • separators(구분 문자)

실제로 입력값 정의는 코드에서 아래처럼 나타납니다.

inputs = [
    IntInput(
        name="chunk_size",
        display_name="Chunk Size",
        info="The maximum length of each chunk.",
        value=1000,
    ),
    IntInput(
        name="chunk_overlap",
        display_name="Chunk Overlap",
        info="The amount of overlap between chunks.",
        value=200,
    ),
    # 이하 생략
]

데이터 처리 로직은 코드에서 아래와 같이 정의됩니다:

    def get_data_input(self) -> Any:
        return self.data_input

    def build_text_splitter(self) -> TextSplitter:
        if not self.separators:
            separators: list[str] | None = None
        else:
            # check if the separators list has escaped characters
            # if there are escaped characters, unescape them
            separators = [unescape_string(x) for x in self.separators]

        return RecursiveCharacterTextSplitter(
            separators=separators,
            chunk_size=self.chunk_size,
            chunk_overlap=self.chunk_overlap,
        )

10. 컴포넌트 버전 관리와 업데이트

각 컴포넌트의 버전 및 상태 정보는 Langflow 내부 데이터베이스에 저장됩니다.

컴포넌트를 플로우에 추가하는 순간, 그 시점의 버전이 고정되며, 이후 Langflow 프로그램을 업그레이드해도 자동으로 바뀌지 않습니다.

즉, 특정 컴포넌트를 플로우에 추가할 당시 버전이 1.0이었다면, 명시적으로 업데이트하지 않는 이상 해당 플로우 내에서는 계속 1.0으로 유지됩니다.

업데이트 알림 유형:

  • Update ready: 안전한 업데이트, 기존 흐름에 영향 없음
  • Update available: 입력/출력 포트 변경 등 호환성 문제 발생 가능

업데이트 방법:

  • Update 버튼으로 개별 컴포넌트를 바로 업데이트
  • Review 버튼을 눌러 전체 변경 사항 확인 후 업데이트(스냅샷 생성 권장)

업데이트 전에는 프로젝트를 백업하는 것이 좋습니다. 업데이트 전 플로우를 백업하려면 "Create backup flow before updating" 옵션을 활성화하세요. 백업은 동일한 프로젝트 폴더에 (backup)이라는 이름으로 저장됩니다.

11. 컴포넌트 그룹화 및 재사용

여러 컴포넌트를 묶어서 하나의 그룹 컴포넌트로 만들 수도 있습니다.

이렇게 하면 복잡한 작업 흐름을 보기 쉽게 정리하고, 재사용할 수도 있습니다.

  • Shift 혹은 Ctrl 키를 누른 채 마우스로 여러 컴포넌트를 선택한 뒤,
  • Group 버튼을 클릭하면 하나로 묶입니다.

이후에는 그룹 전체를 하나의 컴포넌트처럼 관리할 수 있습니다.

그룹 해제가 필요할 때는 Show More → Ungroup 메뉴를 사용하세요.

자주 쓰는 그룹은 컴포넌트 메뉴에 따로 저장해 둘 수도 있습니다.

상단의 프로필 아이콘 → 설정(Settings) → 단축키(Shortcuts)에서 확인할 수 있습니다.
