Processing

프로세싱 (Processing) 컴포넌트

: Components > Processing

Langflow의 Processing 컴포넌트는 플로우 내 데이터를 처리하고 변환합니다.

플로우에서 Processing 컴포넌트 사용하기

Split Text 처리 컴포넌트는 들어오는 Data를 여러 청크(chunk)로 분할하여 벡터 스토어 컴포넌트에 임베딩할 수 있도록 합니다.

청크 크기(chunk size), 중첩(overlap), 구분자(separator)를 조절하여 벡터 스토어에서 검색할 때의 문맥과 세분화 정도를 제어할 수 있습니다.

DataFrame 연산(DataFrame operations)

이 컴포넌트는 DataFrame의 행(row)과 열(column)에 대해 다양한 연산을 수행합니다.

DataFrame Operations 컴포넌트를 사용하려면, DataFrame을 출력하는 컴포넌트를 해당 컴포넌트의 입력에 연결하세요.

아래 예시에서는 API로부터 JSON 데이터를 가져와 Smart function 컴포넌트로 평탄화(flatten)하여 테이블 형식의 DataFrame을 생성한 후, DataFrame Operations 컴포넌트로 필터링 연산을 수행합니다.

  1. API Request 컴포넌트로부터 source와 result 필드만 포함된 데이터를 가져옵니다. 이 예시에서는 실제 데이터가 result 필드 내부에 중첩되어 있습니다.
  1. Smart function 컴포넌트를 API Request에 연결하고, Language model(이 예시에서는 Groq 모델)도 연결하세요.
  1. Groq 모델 컴포넌트에 Groq API 키를 입력합니다.
  1. Smart function 컴포넌트의 Instructions 필드에 다음과 같이 자연어로 입력하여 데이터를 평탄화하세요.
    I want to explode the result column out into a Data object

    Tip: Instructions 필드에 구두점(punctuation)을 포함하면 오류가 발생할 수 있으니 피하십시오.

  1. Smart function 컴포넌트에서 Run component를 클릭하여 흐름을 실행합니다.
  1. Smart function 컴포넌트에서 Inspect output을 클릭하면, 다음과 같은 구조화된 DataFrame을 확인할 수 있습니다.
idnamecompanyusernameemailaddresszip
1Emily JohnsonABC Corporationemily_johnsonemily.johnson@abccorporation.com123 Main St12345
2Michael WilliamsXYZ Corpmichael_williamsmichael.williams@xyzcorp.com456 Elm Ave67890
  1. DataFrame Operations 컴포넌트와 Chat Output 컴포넌트를 플로우에 추가합니다.
  1. DataFrame Operations 컴포넌트의 Operation 필드에서 Filter를 선택합니다.
  1. Column Name 필드에 필터링할 열 이름(예: name)을 입력합니다.
  1. Playground에서 Run Flow를 클릭하면, 지정한 열의 값이 추출됩니다.
name
Emily Johnson
Michael Williams
John Smith
...

Data to DataFrame

이 컴포넌트는 하나 이상의 Data 객체를 DataFrame으로 변환합니다. 각 Data 객체는 결과 DataFrame의 한 행(row)이 되고, .data 속성의 필드는 열(column)이 되며, .text 필드가 있으면 text 열로 배치됩니다.

  1. Data를 출력하는 컴포넌트를 Data to DataFrame 컴포넌트의 입력에 연결합니다. 예시에서는 Webhook 컴포넌트를 연결합니다.
  1. Chat Output 컴포넌트를 Data to DataFrame 컴포넌트의 출력에 연결하여 결과를 확인합니다.
  1. 다음과 같이 Webhook에 POST 요청을 보내 데이터와 텍스트를 전송하세요. YOUR_FLOW_ID를 실제 플로우 ID로 교체합니다:
    curl -X POST "http://localhost:7860/api/v1/webhook/YOUR_FLOW_ID" \
  -H 'Content-Type: application/json' \
  -d '{
    "text": "Alex Cruz - Employee Profile",
    "data": {
      "Name": "Alex Cruz",
      "Role": "Developer",
      "Department": "Engineering"
    }
  }'
  1. Playground에서 플로우를 실행하면, 다음과 같은 DataFrame으로 변환된 결과를 확인할 수 있습니다:

|:-----------------------------|:------------------------------------------------------------------------|
| Alex Cruz - Employee Profile | {'Name': 'Alex Cruz', 'Role': 'Developer', 'Department': 'Engineering'} |
  1. 다른 직원 데이터로 다시 요청을 보내 보세요:
    curl -X POST "http://localhost:7860/api/v1/webhook/YOUR_FLOW_ID" \

  -d '{
    "text": "Kalani Smith - Employee Profile",

Parser

이 컴포넌트는 DataFrame 또는 Data 객체를 템플릿에 따라 텍스트로 포맷팅하며, stringify 모드를 사용해 입력을 문자열로 바로 변환할 수도 있습니다.

템플릿 변수는 Prompt 컴포넌트와 동일하게 중괄호({}) 안에 작성합니다. DataFrame인 경우 열 이름을, Data인 경우 {text}를 사용하세요.

사용 예시: Structured Output과 함께 사용하기

  1. Structured Output 컴포넌트의 DataFrame 출력을 Parser 컴포넌트의 입력으로 연결합니다.
  1. File 컴포넌트를 Structured Output의 Message 입력에 연결합니다.
  1. OpenAI 모델 컴포넌트의 Language Model 출력을 Structured Output의 해당 입력에 연결합니다.
  1. Structured Output 컴포넌트에서 Open Table을 클릭하여 테이블을 만듭니다. 기본적으로 Name, Description, Type, Multiple 행이 제공됩니다.
  1. 파일에서 로드한 데이터에 맞춰 id, name, email 등 필요한 행을 추가하고, 모두 string 타입으로 설정합니다.
  1. Parser 컴포넌트의 Template 필드에 다음과 같은 마크다운 템플릿을 입력합니다:
    # Employee Profile
## Personal Information
- **Name:** {name}
- **ID:** {id}
- **Email:** {email}
  1. Parser 컴포넌트에서 Run component를 클릭하여 실행합니다.
  1. Inspect output을 클릭하면 포맷팅된 텍스트를 확인할 수 있습니다.
  1. 필요하다면 Chat Output 컴포넌트를 연결하고 Playground에서 결과를 확인하세요.

추가 예시: Market Research 템플릿 흐름에서 Structured Output 결과를 Parser로 포맷팅하는 방법을 참고하세요.

Regex Extractor

이 컴포넌트는 정규식(regular expression)을 사용해 텍스트에서 특정 패턴을 추출합니다.

  1. Regex Extractor를 URL 컴포넌트와 Chat Output 컴포넌트에 연결합니다.
  1. Regex Extractor의 패턴 입력 필드에 다음 정규식을 입력하여 Wikipedia 메인 페이지의 “In the News” 섹션 첫 문단을 추출해 보세요:
    In the news\s*\n(.*?)(?=\n\n)

    결과:

    Peruvian writer and Nobel Prize in Literature laureate Mario Vargas Llosa (pictured) dies at the age of 89.

Save to File 컴포넌트

이 컴포넌트는 DataFrame, Data, 또는 Message를 다양한 파일 형식으로 저장합니다.

  1. 플로우에서 DataFrame, Data, 또는 Message를 출력하는 컴포넌트를 Save to File 컴포넌트의 입력에 연결합니다. 아래 예시는 Webhook 컴포넌트를 두 개의 Save to File 컴포넌트에 연결하여 서로 다른 출력 형식을 보여줍니다.
  1. Input Type 필드에서 예상되는 입력 타입을 선택합니다. 이 예시에서는 Webhook의 Data를 입력으로 사용합니다.
  1. File Format 필드에서 저장할 파일 형식을 선택합니다. 이 예시에서는 한 컴포넌트는 .md, 다른 컴포넌트는 .xlsx를 사용합니다.
  1. File Path 필드에 저장할 파일 경로를 입력합니다. 이 예시에서는 ./output/employees.xlsx와 ./output/employees.md를 사용했습니다. 상대 경로와 절대 경로 모두 지원하며, 필요한 디렉터리가 없으면 자동으로 생성됩니다.

    Tip: file_format과 일치하지 않는 확장자를 입력하면, 컴포넌트가 자동으로 올바른 확장자를 파일명 뒤에 붙여줍니다. 예: file_format이 csv일 때 ./output/test.txt를 입력하면 ./output/test.txt.csv로 저장됩니다.

  1. Webhook에 JSON 데이터를 POST 요청으로 전송합니다(YOUR_FLOW_ID를 실제 플로우 ID로 교체).
    curl -X POST "http://localhost:7860/api/v1/webhook/YOUR_FLOW_ID" \
-H 'Content-Type: application/json' \
-d '{
    "Name": ["Alex Cruz", "Kalani Smith", "Noam Johnson"],
    "Role": ["Developer", "Designer", "Manager"],
    "Department": ["Engineering", "Design", "Management"]
}'
  1. 로컬 파일 시스템의 output 디렉터리를 열면, .xlsx와 .md 파일이 생성된 것을 확인할 수 있습니다.
    NameRoleDepartment
    Alex CruzDeveloperEngineering
    Kalani SmithDesignerDesign
    Noam JohnsonManagerManagement

Smart function 컴포넌트

이 컴포넌트는 LLM을 사용해 구조화된 데이터를 필터링하거나 변환하는 Lambda 함수를 생성합니다.

Language Model 컴포넌트와 연결한 뒤 Instructions 필드에 자연어로 지침을 입력하면, 해당 지침에 맞는 함수를 생성해 데이터를 처리합니다.

예시: https://jsonplaceholder.typicode.com/users API에서 JSON 데이터를 가져와, extract emails 지시를 입력하면 이메일 주소 목록을 추출하는 함수를 생성합니다.

매개변수(Parameters)

입력값(Inputs)

이름표시 이름설명
dataData필터링 또는 변환할 구조화된 데이터 객체
llmLanguage Model모델 컴포넌트 연결 포트
filter_instructionInstructionsLambda 함수로 처리할 데이터 지시 (예: 'status'가 'active'인 항목만 포함하도록 필터링)
sample_sizeSample Size대용량 데이터인 경우, 앞뒤에서 샘플링할 문자 수
max_sizeMax Size데이터가 "대용량"으로 간주될 문자 수 임계값

출력값(Outputs)

이름표시 이름설명
filtered_dataFiltered Data필터링 또는 변환된 Data 객체
dataframeDataFrame결과 데이터를 DataFrame으로 반환

Split Text 컴포넌트

이 컴포넌트는 텍스트를 지정한 기준에 따라 청크(chunk)로 분할합니다. 토큰화 및 벡터베이스 임베딩을 위한 데이터 준비에 적합합니다.

  • Chunks 출력: 개별 텍스트 청크 목록 (Data 객체)
  • DataFrame 출력: 메타데이터가 포함된 구조화된 테이블
  1. 텍스트 또는 DataFrame을 출력하는 컴포넌트를 Split Text 컴포넌트의 Data 포트에 연결합니다. (예: URL 컴포넌트)
  1. Separator, Chunk Size, Chunk Overlap 파라미터를 설정합니다.
    • 우선순위: Separator → Chunk Size → Chunk Overlap
    • Separator로 분할한 후, chunk_size보다 긴 세그먼트는 추가로 잘라냅니다.
    • 이후 청크 간 문맥 유지를 위해 chunk_overlap을 적용합니다.
  1. DataFrame 출력에 Chat Output 컴포넌트를 연결하고 Playground에서 Run Flow를 클릭합니다.
  1. 구분자 }, 기준으로 JSON 객체가 분할된 테이블이 출력됩니다.
    {
  "userId": 1,
  "id": 1,
  ...
},
{
  "userId": 2,
  "id": 2,
  ...
},
  1. Separator를 비운 후 다시 실행하면, 50자 청크에 10자 중첩으로 분할된 텍스트가 출력됩니다.
    First chunk:  "title": "Introduction to Artificial Intelligence""
Second chunk: "elligence", "body": "Learn the basics of Artif"
Third chunk:  "s of Artificial Intelligence and its applications"

매개변수(Parameters)

입력값(Inputs)

이름표시 이름설명
data_inputsInput Documents분할할 Data 또는 DataFrame
chunk_overlapChunk Overlap청크 간 중첩 문자 수 (기본값: 200)
chunk_sizeChunk Size각 청크의 최대 문자 수 (기본값: 1000)
separatorSeparator분할 기준 문자열 (기본값: 줄바꿈)
text_keyText Key텍스트 필드로 사용할 키 (기본값: text)

출력값(Outputs)

이름표시 이름설명
chunksChunks분할된 텍스트 청크 목록 (Data)
dataframeDataFrame분할된 텍스트 청크 테이블 (DataFrame)

Combine text

중요: Legacy 컴포넌트입니다.

두 개의 텍스트를 지정한 구분자로 연결합니다.

  1. 두 개의 Message 출력 컴포넌트를 Combine Text 컴포넌트의 First Text／Second Text에 연결합니다.
  1. 각 Text Input에 텍스트를 입력합니다.
  1. Delimiter 필드에 구분자(예: \n\n **end first text** \n\n **start second text**)를 입력합니다.
  1. Chat Output을 연결하고 Playground에서 Run Flow를 클릭하면 결과를 확인할 수 있습니다.
This is the first text. Let's combine text!
end first text
start second text
Here's the second part. We'll see how combining text works.

매개변수

입력값

이름표시 이름설명
first_textFirst Text첫 번째 텍스트
second_textSecond Text두 번째 텍스트
delimiterDelimiter텍스트 사이에 삽입할 문자열 (기본값: 공백)

출력값

이름표시 이름설명
messageMessage결합된 텍스트를 담은 Message
