Langflow 개요

이 페이지는 Langflow의 기본 구성 요소를 탐색하며, "플로우(flow)란 무엇인가?"라는 질문에서 시작합니다.

플로우란 무엇인가?

  • 플로우(flow)는 애플리케이션의 기능적 표현입니다. 간단히 말해 입력을 받아 처리하고 출력을 생성하는 단위라고 보시면 됩니다.
  • 플로우는 워크스페이스에서 구성하고 연결하는 컴포넌트(components)로 구성합니다.

플로우는 위 사진과 같이 OpenAI 챗봇을 4개의 컴포넌트로 만드는 '기본 프롬프팅 플로우(basic prompting flow)'처럼 간단하게 구성할 수도 있습니다.

  • 플로우의 각 컴포넌트는 AI 모델이나 데이터 소스와 같은 특정 작업을 수행하는 노드(Node)라고 보면 됩니다.
  • 각 컴포넌트에는 구성(Configuration) 메뉴가 있습니다. 컴포넌트의 코드(<>) 버튼을 클릭하면 기본 Python 코드를 볼 수 있습니다.
  • 컴포넌트는 엣지(edges)로 연결되어 플로우를 형성합니다.

React Flow에 익숙하다면, 플로우는 노드 기반 애플리케이션이고, 컴포넌트는 노드이며, 컴포넌트 간의 연결은 엣지라고 보면 됩니다.

플로우의 내부 작동 원리

  1. 플로우가 실행되면 Langflow는 노드(컴포넌트)와 엣지(컴포넌트 간 연결)로부터 방향성 비순환 그래프(DAG) 객체를 구축하고, 실행 순서를 결정하기 위해 노드를 정렬합니다.
  1. 그래프 빌드는 개별 컴포넌트의 def_build 함수를 호출하여 노드를 검증하고 준비합니다. 이 그래프는 의존성 순서대로 처리됩니다.
  1. 각 노드는 순차적으로 구축되고 실행되며, 각 구축된 노드의 결과는 이전 노드의 결과에 의존하는 노드로 전달됩니다.

플로우는 다음과 같은 기본 위치의 로컬 디스크에 저장됩니다:

  • Linux 및 WSL: home/<username>/.cache/langflow/
  • macOS: /Users/<username>/Library/Caches/langflow/

플로우 저장 위치는 LANGFLOW_CONFIG_DIR 환경 변수로 사용자 지정할 수 있습니다.

Langflow 둘러보기

Langflow를 처음 사용한다면 처음에는 약간 혼란스러울 수 있습니다. 여기서 투어를 통해 빠르게 방향을 잡고 애플리케이션을 만들기 시작할 수 있도록 안내해 드리겠습니다.

Langflow에는 네 개의 구별된 영역이 있습니다: 워크스페이스(workspace)는 플로우를 구축하는 주요 영역입니다. 왼쪽에 있는 컴포넌트(components) 메뉴는 사용 가능한 컴포넌트를 나열합니다. 플레이그라운드(playground)와 퍼블리시 패널(publish pane)은 오른쪽 상단 모서리에서 사용할 수 있습니다.

워크스페이스

워크스페이스는 플로우에서 컴포넌트를 연결하고 실행하여 AI 애플리케이션을 만드는 작업 공간입니다.

  • 워크스페이스를 클릭하고 드래그하여 좌우상하로 이동할 수 있습니다.
  • 위아래로 스크롤하여 워크스페이스를 확대하거나 축소하거나, 확대 및 축소 컨트롤을 사용할 수 있습니다.
  • 화면에 맞추기(Fit To Zoom)를 클릭하여 현재 플로우에 워크스페이스를 중앙에 배치합니다.
  • 잠금(Lock)을 클릭하여 워크스페이스를 제자리에 고정하고 실수로 이동하는 것을 방지합니다.
  • 메모 추가를 클릭하여 코드 주석과 유사하게 플로우에 메모를 추가합니다.

컴포넌트

컴포넌트는 플로우의 기본 구성 요소입니다. 자세한 내용은 컴포넌트 개요(Components overview)를 참조하세요.

플레이그라운드(Playground)

  • 플레이그라운드는 워크스페이스의 현재 플로우를 실행합니다.
  • 플로우와 채팅하고, 입력과 출력을 보고, AI의 메모리를 수정하여 실시간으로 응답을 조정할 수 있습니다.
  • 채팅 입력 또는 채팅 출력 컴포넌트는 플레이그라운드( Playground)에서 열고 실시간으로 테스트할 수 있습니다.

퍼블리시 패널

퍼블리시 패널( Publish pane)은 만든 플로우를 외부에서 사용할 수 있도록 배포(publish) 하는 기능을 제공합니다. 아래와 같이 외부 애플리케이션에 플로우를 통합하기 위한 코드 템플릿을 제공합니다.

🔹 퍼블리시 패널의 주요 용도

  1. 공유 가능한 링크 생성
    • 만든 플로우를 외부 사용자에게 웹 인터페이스로 제공할 수 있습니다.
    • 사용자는 로그인 없이도 해당 링크를 통해 LangChain 기반 챗봇이나 앱을 사용할 수 있습니다.
  1. 간단한 웹 UI 인터페이스 제공
    • 퍼블리시된 플로우는 기본적으로 Langflow에서 제공하는 입력 창 / 응답창 UI 형태로 나타납니다.
    • 사용자는 노드 연결을 몰라도 입력값을 넣고 결과를 확인할 수 있습니다.
  1. API 호출이 아닌, 사용자 친화적인 접근 제공
    • LangChain 기반 워크플로우를 서버 없이 빠르게 테스트 / 데모 / 배포할 수 있는 경로입니다.
  1. 팀원/클라이언트에게 시연하거나 피드백 받기 좋음
    • 복잡한 코드나 백엔드 서버 구성 없이도 클라이언트에게 공유해서 쉽게 피드백을 받을 수 있습니다.

로그 보기

  • 로그 패널은 워크스페이스 내 모든 컴포넌트 실행에 대한 자세한 기록을 제공합니다.
  • 로그 패널에 액세스하려면 플로우 이름을 클릭한 다음 로그를 선택합니다.

Langflow는 LANGFLOW_CONFIG_DIR 환경 변수에 지정된 위치에 로그를 저장합니다.

이 디렉토리의 기본 위치는 운영 체제에 따라 다릅니다.

  • Linux 및 WSL: ~/.cache/langflow/
  • macOS: /Users/<username>/Library/Caches/langflow/
  • Windows: %LOCALAPPDATA%\langflow\langflow\Cache

로그 파일 위치를 수정하려면:

  1. LANGFLOW_LOG_FILE=path/to/logfile.log를 .env 파일에 추가합니다.
  1. .env 파일의 값으로 Langflow를 시작하려면 uv run langflow run --env-file .env로 Langflow를 시작합니다.

예제 .env 파일은 프로젝트 저장소에서 확인할 수 있습니다.

프로젝트

프로젝트 페이지는 Langflow 워크스페이스에서 생성한 모든 플로우를 표시합니다.

스타터 프로젝트는 모든 새 프로젝트가 처음에 저장되는 기본 공간입니다. 새 프로젝트를 만들려면 새 프로젝트 만들기(+)를 클릭합니다.

프로젝트에 플로우를 업로드하려면 플로우 업로드를 클릭합니다.

프로젝트에서 플로우를 삭제하려면 플로우의 체크박스를 클릭하여 선택한 다음 삭제를 클릭합니다. 한 번의 작업으로 여러 플로우를 선택할 수 있습니다.

파일 관리( Manage files)

Langflow의 파일 관리 시스템에서 파일을 업로드, 저장 및 관리합니다.

옵션 메뉴

프로젝트 이름으로 표시된 드롭다운 메뉴는 Langflow 워크스페이스의 현재 플로우에 대한 여러 관리 및 사용자 지정 옵션을 제공합니다:

  • 새로 만들기: 처음부터 새 플로우를 만듭니다.
  • 세부 정보 편집: 이름, 설명, 엔드포인트 이름과 같은 현재 플로우에 특정한 설정을 조정합니다.
  • 로그: 실행 기록, 오류 및 기타 런타임 이벤트를 포함한 현재 프로젝트의 로그를 봅니다.
  • 가져오기: JSON 파일에서 플로우나 컴포넌트를 워크스페이스로 가져옵니다.
  • 내보내기: 현재 플로우를 JSON 파일로 내보냅니다.
  • 실행 취소: 프로젝트에서 마지막으로 수행한 작업을 되돌립니다. 키보드 단축키: Control+Z(macOS에서는 Command+Z).
  • 다시 실행: 이전에 실행 취소한 작업을 다시 적용합니다. 키보드 단축키: Control+Y(macOS에서는 Command+Y).
  • 모두 새로 고침: 모든 컴포넌트를 새로 고치고 캐시를 삭제합니다.

설정

Langflow 헤더에서 프로필 아이콘을 클릭한 다음 설정을 선택하여 전역 변수, Langflow API 키, 키보드 단축키 및 로그 메시지를 포함한 일반 Langflow 설정에 액세스합니다.
