Tools

툴 (Tool) 컴포넌트

: Components > Tools

Langflow의 툴 (Tool) 컴포넌트는 일반적으로 Agent 컴포넌트의 Tools 포트에 연결됩니다. Agent는 LLM을 추론 엔진으로 사용하여, 연결된 Tool 컴포넌트 중 어떤 도구를 사용해야 문제를 해결할지 결정합니다.

툴이란?

툴은 쉽게 말해 Agent가 호출하여 특정 작업을 수행하거나 외부 리소스에 접근할 수 있는 함수(function)입니다. 각 함수는 Agent가 이해할 수 있는 공통 인터페이스를 가진 툴 객체로 래핑(wrap)됩니다. Agent는 시작할 때 사용 가능한 Tool 목록을 받아 이를 통해 어떤 도구가 있는지 인식하며, 각 툴 객체의 설명(description)을 참고해 해당 도구가 수행할 수 있는 작업을 이해합니다. 이후 Agent는 연결된 LLM을 사용해 문제를 추론하고, 가장 적합한 도구를 선택해 호출합니다.

플로우에서 Tool 사용하기

Tool은 Agent 컴포넌트의 Tools 포트에 연결하여 사용합니다.

간단한 Agent 시작 프로젝트(simple agent starter project)에서는 URL과 Calculator 도구를 Agent 컴포넌트에 연결해 사용자의 질문에 답변하도록 구성되어 있습니다. 이때 OpenAI LLM이 Agent의 두뇌 역할을 수행하며, 어떤 도구를 사용할지 결정합니다.

어떤 컴포넌트를 Agent에서 사용할 수 있는 도구로 만들려면, 해당 컴포넌트에서 Tool Mode 옵션을 활성화하면 됩니다. Tool Mode를 활성화하면 컴포넌트의 입력(input)이 Agent 호출을 수락하도록 변경됩니다. 만약 연결하려는 컴포넌트에 Tool Mode 옵션이 없다면, 직접 입력 구성을 수정해 도구로 사용할 수 있습니다. 자세한 예시는 “Make any component a tool” 문서를 참조하세요.

Astra DB Tool

이 컴포넌트는 Agent가 Astra DB 컬렉션에서 데이터를 조회할 수 있도록 합니다.

플로우 예시:

  • Tool Name과 Tool Description은 필수 입력이며, Tool Name에는 공백을 사용할 수 없습니다.
  • Collection Name, Astra DB Application Token, Astra DB API Endpoint는 Astra DB 배포 환경에서 확인할 수 있습니다(자세한 내용은 DataStax 문서 참조).

예시에서는 OpenAI Embeddings 컴포넌트를 Astra DB Tool의 Semantic Search 기능과 연결했습니다. Semantic Search를 사용하려면 임베딩 모델 또는 Astra DB Vectorize가 활성화되어 있어야 하며, 임베딩 모델이 없으면 오류가 발생합니다. Playground에서 질문을 입력하면 Agent가 Astra DB Tool을 통해 컬렉션 정보를 반환합니다.

Astra DB Tool 파라미터 설정

Tool Parameters 탭에서 Find 명령에 적용할 필터 파라미터를 정의할 수 있으며, LLM은 이 파라미터 정보를 바탕으로 도구를 호출할 때 적절한 값을 제공합니다.

  1. Add a new row 클릭
  1. 다음 예시처럼 설정:
    • Name: customer_id
    • Attribute Name: 실제 필드명과 동일하면 비워둡니다.
    • Description: 조회할 고객의 고유 식별자
    • Is Metadata: 메타데이터 필드일 경우 켜기
    • Is Mandatory: 필수 필터일 경우 켜기
    • Is Timestamp: 날짜/시간 필터가 아니면 끄기
    • Operator: $eq (정확 일치)

LLM 입력과 무관하게 필터를 고정하려면 Static Filters 옵션을 사용하세요.

파라미터설명
NameLLM에 노출될 파라미터 이름. 실제 필드명과 일치하거나 더 설명적인 이름 사용 가능.
Attribute NameLLM 이름과 실제 DB 속성이 다를 때 매핑 설정. 예: start_date, end_date로 같은 timestamp 필드 참조.
DescriptionLLM이 파라미터를 어떻게 사용해야 하는지 안내. 예: specialty를 소문자로 입력해야 한다고 명시.
Is Metadatametadata 객체 하위에 저장된 속성일 때 켜기. 쿼리 시 {"metadata.<attribute_name>":"<value>"} 형태로 처리됩니다.
Is Timestamp날짜/시간 필터일 때 켜기. Astrapy 클라이언트가 기대하는 타임스탬프 형식으로 자동 변환해줍니다.
Operator필터 연산자 지정. 예: $gt(크다), $lt(작다) 등 유효한 Data API 필터 연산자 사용.

Parameters

Inputs:

이름타입설명
Tool NameStringAgent 프롬프트에서 사용될 도구 이름
Tool DescriptionString도구 설명 (모델이 언제 사용해야 할지 결정)
Collection NameString조회할 Astra DB 컬렉션 이름
TokenSecretStringAstra DB 접근을 위한 인증 토큰
API EndpointStringAstra DB API 엔드포인트
Projection FieldsString반환할 속성(콤마 구분), 기본값: \*
Tool ParametersDict모델이 도구 실행에 채울 파라미터 (필수 파라미터는 느낌표(!) 사용)
Static FiltersDict쿼리 결과 필터용 속성-값 쌍
LimitString반환할 문서 수

Outputs:

이름타입설명
DataList\[Data]Projection Fields로 지정한 필드를 포함한 쿼리 결과 Data 객체 목록 (number\_of\_results 제한)
ToolStructuredToolLangChain StructuredTool 객체 (파라미터 스키마·쿼리 함수 포함)
