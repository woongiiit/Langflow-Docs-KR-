Embeddings

임베딩 (Embedding) 컴포넌트

: Components > Embeddings

Langflow의 임베딩(Embeddings) 컴포넌트는 텍스트를 숫자 형태의 **벡터(vector)**로 변환하는 역할을 합니다. 이 벡터는 텍스트의 의미적, 문맥적 의미를 담고 있어, 대규모 언어 모델(LLM)이 복잡한 데이터를 이해하고 처리할 수 있게 돕는 핵심 기능입니다.

1. 플로우에서 임베딩 컴포넌트 사용하기

임베딩 컴포넌트는 주로 데이터 수집 파이프라인에서 사용됩니다. 예를 들어, OpenAI Embeddings 컴포넌트를 벡터 데이터베이스(예: Astra DB)에 연결하면, 텍스트 덩어리(chunks)가 벡터로 변환되어 데이터베이스에 저장됩니다.

이렇게 벡터화된 데이터는 챗봇, 유사성 검색, RAG 에이전트 등 다양한 AI 워크로드에 활용될 수 있습니다.

예시) 문서 벡터화 및 저장 파이프라인

아래는 OpenAI Embeddings 컴포넌트를 Astra DB 벡터 스토어에 연결하여 텍스트 데이터를 임베딩하고 저장하는 파이프라인 예시입니다.

4. Astra DB vectorize

중요

이 컴포넌트는 Langflow 버전 1.1.2부터 더 이상 사용되지 않습니다. 대신 Astra DB 벡터 스토어 컴포넌트를 사용하세요.

Astra DB 벡터 스토어의 Embeddings 포트에 연결하여 임베딩을 생성합니다. 사용하려면 Astra DB에 vectorize 임베딩 제공자 통합이 활성화된 컬렉션이 필요합니다.

입력값 (Inputs)

이름표시 이름정보
providerEmbedding Provider사용할 임베딩 제공자입니다.
model_nameModel Name사용할 임베딩 모델입니다.
authenticationAuthenticationvectorize 제공자 자격 증명을 저장한 Astra API 키 이름입니다.
provider_api_keyProvider API Keyauthentication 대신 API 키를 직접 제공할 수 있습니다.
model_parametersModel Parameters추가적인 모델 파라미터입니다.

출력값 (Outputs)

이름타입설명
embeddingsEmbeddingsAstra vectorize를 사용해 임베딩을 생성하는 인스턴스입니다.

10. Hugging Face Embeddings

참고

이 컴포넌트는 Langflow 버전 1.0.18부터 더 이상 사용되지 않습니다. 대신 Hugging Face Embeddings Inference 컴포넌트를 사용하세요.

로컬에 다운로드된 HuggingFace 모델로 임베딩을 생성합니다. 모델을 가동하기에 충분한 컴퓨팅 리소스가 있는지 확인하시기 바랍니다.

이름표시 이름정보

출력값 (Outputs)

이름표시 이름정보

11. Hugging Face embeddings inference

Hugging Face Inference API 또는 로컬 호스팅 모델을 사용하여 임베딩을 생성합니다. Inference API 사용 시 API 토큰이 필요합니다.

로컬 모델 연결 및 사용법

  1. 벡터 스토어 RAG 플로우를 생성하고, 기존 OpenAI Embeddings 컴포넌트를 Hugging Face embeddings inference 컴포넌트로 교체합니다.
  1. Hugging Face 컴포넌트를 벡터 스토어(예: Astra DB)의 Embeddings 포트에 연결합니다.
  1. Inference Endpoint 필드에 로컬 추론 모델의 URL을 입력합니다. (로컬 추론 시 API Key는 필요 없음)
  1. 플로우를 실행하여 로컬 모델로 임베딩을 생성합니다.

입력값 (Inputs)

이름표시 이름정보
API KeyAPI KeyHugging Face Inference API에 접근하기 위한 API 키입니다.
API URLAPI URLHugging Face Inference API의 URL입니다.
Model NameModel Name임베딩에 사용할 모델의 이름입니다.

출력값 (Outputs)

이름표시 이름정보
embeddingsEmbeddings생성된 임베딩입니다.
