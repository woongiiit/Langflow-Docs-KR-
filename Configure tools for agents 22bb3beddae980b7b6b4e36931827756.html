에이전트용 도구 구성

에이전트의 기능을 확장하기 위해 연결된 도구들을 구성합니다.

도구 컴포넌트의 동작 편집

도구의 동작을 편집하려면, 도구 컴포넌트에서 도구 편집(Edit Tools)을 클릭하여 해당 name (이름), description (설명), 또는 enabled (활성화) 메타데이터를 수정할 수 있습니다. 이 필드는 연결된 에이전트가 도구를 어떻게 사용하는지 이해하는 데 도움을 줍니다. 에이전트의 프롬프트(입력 지침)를 수정하지 않고도 도구를 이해할 수 있도록 돕습니다.

예를 들어, URL 컴포넌트는 Tool Mode가 활성화되면 두 가지 동작을 수행할 수 있습니다:

도구 이름설명활성화 여부
fetch_content웹 페이지에서 내용을 재귀적으로 가져오기true
fetch_content_as_message웹 콘텐츠를 메시지 형식으로 가져오기true

Langflow 에이전트는 각 도구의 name (이름)과 description (설명) 메타데이터를 기반으로 도구의 기능을 명확하게 이해할 수 있습니다. enabled (활성화) 값은 도구가 에이전트에게 사용 가능한지 여부를 제어합니다. 에이전트가 도구를 잘못 사용한다고 생각되면, 도구의 설명 메타데이터를 수정하여 에이전트가 도구를 더 잘 이해하도록 도울 수 있습니다.

도구 이름과 설명은 편집할 수 있지만, 기본 도구 식별자(identifier)는 변경할 수 없습니다. 도구 식별자를 변경하려면, 사용자 정의 컴포넌트를 만들어야 합니다.

에이전트를 도구로 사용하기

에이전트 컴포넌트 자체도 Tool Mode를 지원하여 멀티 에이전트 흐름을 만들 수 있습니다.

  1. 단순 에이전트 시작 흐름을 생성합니다.
  1. 두 번째 에이전트 컴포넌트를 흐름에 추가합니다.
  1. 에이전트 컴포넌트에 Open AI API Key를 추가합니다.
  1. 모델 이름 필드에서 gpt-4.1을 선택합니다.
  1. Tool Mode를 클릭하여 새로운 에이전트를 도구로 사용합니다.
  1. 새로운 에이전트의 Toolset 포트를 이전에 생성한 에이전트의 Tools 포트에 연결합니다. 이렇게 하면 새로운 에이전트가 gpt-4.1 모델을 사용하여 큰 맥락을 요구하는 정보 검색 및 스크래핑을 처리하고, 이전에 생성한 에이전트는 이 에이전트를 도구로 사용하게 됩니다.
  1. 새로운 에이전트의 동작을 수정하여 에이전트가 이를 어떻게 사용할지 이해할 수 있도록 합니다. **도구 편집(Edit Tools)**을 클릭하여 name (이름), description (설명), 또는 enabled (활성화) 메타데이터를 수정합니다. 예를 들어 기본 도구 이름은 Agent이지만 이를 Agent-gpt-41으로 수정하고 설명을 복잡한 문제 해결을 위해 gpt-4.1 모델을 사용합니다로 수정할 수 있습니다. 이렇게 하면 연결된 에이전트는 이것이 gpt-4.1 에이전트임을 이해하고, 더 큰 맥락 창을 필요로 하는 작업에 이를 사용하게 됩니다.

사용자 정의 컴포넌트를 도구로 추가하기

에이전트는 사용자 정의 컴포넌트를 도구로 사용할 수 있습니다.

  1. 사용자 정의 컴포넌트를 에이전트 흐름에 추가하려면 새 사용자 정의 컴포넌트(New Custom Component)를 클릭합니다.
  1. 사용자 정의 Python 코드를 사용자 정의 컴포넌트의 Code (코드) 창에 추가합니다. 예를 들어, 텍스트 분석기 컴포넌트를 만들려면 아래의 코드를 추가합니다.
python
복사편집
from langflow.custom import Component
from langflow.io import MessageTextInput, Output
from langflow.schema import Data
import re

class TextAnalyzerComponent(Component):
    display_name = "Text Analyzer"
    description = "입력된 텍스트를 분석하고 변환합니다."
    documentation: str = "http://docs.langflow.org/components/custom"
    icon = "chart-bar"
    name = "TextAnalyzerComponent"

    inputs = [
        MessageTextInput(
            name="input_text",
            display_name="입력 텍스트",
            info="분석할 텍스트를 입력하세요.",
            value="Hello, World!",
            tool_mode=True,
        ),
    ]

    outputs = [
        Output(display_name="분석 결과", name="output", method="analyze_text"),
    ]

    def analyze_text(self) -> Data:
        text = self.input_text
        # 텍스트 분석
        word_count = len(text.split())
        char_count = len(text)
        sentence_count = len(re.findall(r'\w+[.!?]', text))
        # 텍스트 변환
        reversed_text = text[::-1]
        uppercase_text = text.upper()

        analysis_result = {
            "original_text": text,
            "word_count": word_count,
            "character_count": char_count,
            "sentence_count": sentence_count,
            "reversed_text": reversed_text,
            "uppercase_text": uppercase_text
        }

        data = Data(value=analysis_result)
        self.status = data
        return data
  1. 사용자 정의 컴포넌트를 도구로 사용하려면 Tool Mode를 클릭합니다.
  1. 사용자 정의 컴포넌트의 도구 출력을 에이전트의 도구 입력에 연결합니다.
  1. Playground에서 에이전트에게 이 텍스트에 대해 텍스트 분석기를 사용하세요: "Agents really are thinking machines!"라고 요청합니다.

모든 컴포넌트를 도구로 만들기

도구 모드 버튼이 없는 컴포넌트를 도구로 사용하려면, 컴포넌트 입력 중 하나에 tool_mode=True를 추가하고, 새로 생성된 Toolset 출력을 에이전트의 Tools 입력에 연결합니다.

Langflow는 다음 데이터 유형에 대해 Tool Mode를 지원합니다:

  • DataInput
  • DataFrameInput
  • PromptInput
  • MessageTextInput
  • MultilineInput
  • DropdownInput

예를 들어, 위의 사용자 정의 컴포넌트를 도구로 추가하기 예시에서는 MessageTextInput 입력에 tool_mode=True를 추가하여 사용자 정의 컴포넌트를 도구로 사용할 수 있게 했습니다.

inputs = [
    MessageTextInput(
        name="input_text",
        display_name="입력 텍스트",
        info="분석할 텍스트를 입력하세요.",
        value="Hello, World!",
        tool_mode=True,
    ),
]

흐름을 도구로 사용하기

에이전트는 워크스페이스에 저장된 흐름을 도구로 사용할 수 있습니다. 이를 위해 Run flow 컴포넌트를 사용합니다.

  1. Run flow 컴포넌트를 클릭하고 드래그하여 워크스페이스에 추가합니다.
  1. 에이전트가 사용할 흐름을 선택합니다.
  1. 컴포넌트에서 Tool Mode를 활성화합니다. 그러면 선택한 흐름이 사용 가능한 동작으로 표시됩니다.
  1. Run flow 컴포넌트의 도구 출력을 에이전트의 도구 입력에 연결합니다.
  1. 에이전트에게 내가 질문에 답하기 위해 사용하는 도구는 무엇인가요?라고 묻습니다. 연결된 흐름이 도구로 표시됩니다.
  1. 에이전트에게 연결된 도구를 사용하여 질문에 답하도록 요청
