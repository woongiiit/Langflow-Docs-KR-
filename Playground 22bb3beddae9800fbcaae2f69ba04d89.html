Playground

Playground는 LLM과 실시간으로 상호작용할 수 있도록 설계된 동적 인터페이스로, 사용자가 채팅하고, 메모리에 접근하며, 입력과 출력을 수행할 수 있게 해줍니다. 여기서 사용자는 자신의 모델을 직접 프로토타이핑하고, 조정하며 다양한 결과를 관찰할 수 있습니다.

입력 또는 출력 컴포넌트가 작동하는 한, Playground 버튼을 클릭하여 열 수 있습니다. Playground의 창 배열은 사용 중인 컴포넌트에 따라 달라집니다.

Playground 실행 시 어떤 백엔드 함수들이 호출되는가?

  • Langflow를 커스터마이징하거나 확장하려는 개발자, 새로운 컴포넌트를 직접 만들거나, 기존 동작 방식을 디버깅하려는 사람, 백엔드 API를 직접 호출하거나 통합하러는 개발자 아닌 이상 이 문단은 생략하고 넘어가셔도 됩니다.

Playground에서 flow를 실행하면, Langflow는 chat.py의 /build/$FLOW_ID/flow 엔드포인트를 호출합니다. 이 호출은 flow 데이터를 검색하고, 그래프를 구축하여 실행합니다. 각 컴포넌트(또는 노드)가 실행될 때, build_vertex 함수는 build_and_run을 호출하며, 이는 개별 컴포넌트의 def_build 메서드를 호출할 수 있습니다(존재하는 경우). 컴포넌트에 def_build 함수가 없더라도 빌드는 여전히 컴포넌트를 반환합니다.

build 함수는 컴포넌트가 런타임에 로직을 실행할 수 있게 합니다. 예를 들어, 재귀적 문자 텍스트 분할기는 LCTextSplitterComponent 클래스의 자식입니다. 텍스트를 처리해야 할 때, 부모 클래스의 build 메서드가 호출되어 RecursiveCharacterTextSplitter 객체를 생성하고 정의된 매개변수에 따라 텍스트를 분할합니다. 분할된 텍스트는 다음 컴포넌트로 전달됩니다. 이 모든 과정은 컴포넌트가 빌드될 때 발생합니다.

이 내용은 Langflow를 커스터마이징하거나 확장하려는 개발자, 새로운 컴포넌트를 직접 만들거나, 기존 동작 방식을 디버깅하려는 사람, 백엔드 API를 직접 호출하거나 통합하려는 개발자

세션 ID로 Playground 메시지 보기

Playground 인터페이스에서 메시지를 보내면, 대화내역들은 session_id로 메시지 로그에 저장됩니다. 하나의 flow는 여러 채팅을 가질 수 있으며, 다른 flow들이 같은 채팅을 공유할 수 있습니다. 각 채팅은 서로 다른 session_id를 갖게 됩니다.

Playground 내에서 session_id별로 메시지를 보려면, 채팅 세션의 옵션 메뉴를 클릭한 다음 메시지 로그를 선택하세요.

채팅 메모리의 개별 메시지는 편집하거나 삭제할 수 있습니다. 이러한 메모리를 수정하면 챗봇 응답의 동작에 영향을 미칩니다.

Langflow의 채팅 메모리에 대해 자세히 알아보려면 메모리 컴포넌트를 참조하세요.

다중 사용자 상호작용을 위한 커스텀 세션 ID 사용

session_id 값은 flow에서 사용자 상호작용을 추적하는 데 사용됩니다. 기본적으로 session_id 값이 비어 있으면, flow_id와 동일한 값으로 설정됩니다. 이 경우 모든 채팅 호출은 동일한 session_id를 사용하며, 효과적으로 하나의 채팅 세션을 갖게 됩니다.

session_id 값은 채팅 입력과 채팅 출력 컴포넌트의 고급 설정에서 구성할 수 있습니다.

단일 flow에서 둘 이상의 세션을 갖기 위해, URL의 session_id 매개변수를 사용하여 특정 세션 ID를 flow에 전달할 수 있습니다. flow의 모든 컴포넌트는 자동으로 이 session_id 값을 사용합니다.

curl을 사용하여 특정 세션 ID로 flow에 메시지를 게시하려면 다음 명령을 입력하세요:

curl -X POST "http://localhost:7860/api/v1/run/$FLOW_ID" \
-H 'Content-Type: application/json' \
-d '{
    "session_id": "custom_session_123",
    "input_value": "message",
    "input_type": "chat",
    "output_type": "chat"
}'

flow의 Playground를 확인하세요. 기본 세션에 저장된 메시지 외에도 커스텀 세션 ID로 새 세션이 시작됩니다.

Playground에서 이미지 작업하기

Playground는 base64 형식의 이미지 처리를 지원하여 flow에서 직접 이미지 데이터로 작업할 수 있게 해줍니다.

Playground는 다음과 같은 이미지 형식을 허용합니다:

  • PNG
  • JPG/JPEG
  • GIF
  • BMP
  • WebP

Playground에서 base64 이미지로 작업하는 방법은 여러 가지가 있습니다:

  • 직접 업로드: 채팅 인터페이스의 이미지 업로드 버튼을 사용하여 이미지를 직접 업로드합니다.
  • 드래그 앤 드롭: 이미지 파일을 채팅 인터페이스로 드래그 앤 드롭합니다.
  • 프로그래매틱 입력: API를 통해 base64로 인코딩된 이미지를 전송합니다.

다음 예제는 curl을 사용하여 base64로 인코딩된 이미지를 Playground에 전송합니다:

curl -X POST "http://localhost:7860/api/v1/run/$FLOW_ID" \
-H 'Content-Type: application/json' \
-d '{
    "session_id": "custom_session_123",
    "input_value": "What is in this image?",
    "input_type": "chat",
    "output_type": "chat",
    "files": ["data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA..."]
}'

이미지는 채팅 인터페이스에 표시되며 flow 컴포넌트에 의해 처리될 수 있습니다.
