음성 모드

Langflow Playground는 마이크를 통해 애플리케이션과 상호작용할 수 있는 음성 모드를 지원합니다.

음성 모드를 사용하기 위해서는 OpenAI API 키가 필요합니다. ElevenLabs API 키를 사용하면 채팅에서 더 다양한 음성을 사용할 수 있지만, 이는 선택사항입니다.

Playground와 상호작용하기 위해서는 플로우에 채팅 입력 컴포넌트가 포함되어 있어야 합니다.

필수 조건

  • OpenAI API 키

Langflow Playground에서 음성 모드 사용하기

Playground에서 에이전트와 대화하고, 에이전트에게 도구를 사용하도록 요청하여 더 최신 결과를 얻을 수 있습니다.

  1. 간단한 에이전트 스타터 프로젝트를 생성합니다.
  1. 에이전트 컴포넌트에 OpenAI API 키 자격 증명을 추가합니다.
  1. 채팅 세션을 시작하려면 Playground를 클릭합니다.
  1. 음성 모드를 활성화하려면 마이크 아이콘을 클릭합니다. 음성 모드 패널이 열립니다.
  1. OpenAI API 키 필드에 OpenAI API 키 자격 증명을 추가합니다. 이 키는 Langflow에서 전역 변수(모든 컴포넌트나 플로우에서 접근 가능한 변수)로 저장되어 모든 컴포넌트나 플로우에서 접근할 수 있습니다.
  1. 브라우저에서 마이크 접근 권한을 요청할 수 있습니다. 음성 모드를 사용하려면 브라우저 접근 권한이 필수입니다. 계속하려면 브라우저에서 마이크 접근을 허용하세요.
  1. 오디오 입력 메뉴에서 음성 모드에 사용할 입력 장치를 선택합니다.

팁

고품질 마이크를 사용하면 OpenAI의 음성 채팅 이해도가 향상됩니다.

  1. 선택적으로 ElevenLabs API 키 필드에 ElevenLabs API 키를 추가합니다. 이렇게 하면 AI 응답에 더 많은 음성을 사용할 수 있습니다. 이 키는 Langflow에서 전역 변수로 저장되어 모든 컴포넌트나 플로우에서 접근할 수 있습니다.
  1. 선호 언어 메뉴에서 Langflow와 대화할 언어를 선택합니다. 이 옵션은 음성 대화와 Playground의 채팅 응답 모두를 변경합니다.
  1. 마이크에 대고 말합니다. 음성 모드 패널의 파형(파동 그래프)이 입력을 감지하고, 에이전트가 음성과 Playground에서 응답해야 합니다.
  1. 에이전트에게 특정 주제에 관한 최신 뉴스를 찾기 위해 사용 가능한 도구를 사용하도록 요청합니다.

에이전트는 최신 뉴스를 가져오기 위해 URL 도구에 접근하는 것을 포함한 검색 프로세스를 설명합니다. 에이전트는 최신 뉴스를 음성과 Playground에서 요약합니다.

음성 모드를 사용할 때 다음 사항에 유의하세요:

  • 도구의 이름과 설명을 정확하게 지정하여 에이전트가 도구를 올바르게 선택하도록 합니다.
  • 음성 모드는 에이전트 컴포넌트의 에이전트 지침 필드에 있는 지침을 사용하지 않습니다. 왜냐하면 사용자의 음성 지침이 이 값을 덮어쓰기 때문입니다.
  • 음성 모드는 현재 대화 세션 내에서만 컨텍스트(대화 맥락)를 유지합니다. 대화를 종료하고 Playground를 닫으면, 다음 채팅 세션에서는 대화 컨텍스트를 사용할 수 없습니다.

Langflow 음성 모드 엔드포인트

Langflow는 플로우를 위한 OpenAI 실시간 API 호환 웹소켓 엔드포인트(시스템 간 통신 지점)를 제공합니다. OpenAI 실시간 API 웹소켓을 구축하는 것과 동일한 방식으로 이러한 엔드포인트에 대해 음성 애플리케이션을 구축할 수 있습니다.

웹소켓 엔드포인트는 인증을 위해 OpenAI API 키가 필요하며, 선택적으로 ElevenLabs 통합을 지원합니다.

Langflow는 두 가지 웹소켓 엔드포인트를 제공합니다:

  • /ws/flow_as_tool/{flow_id} 또는 /ws/flow_as_tool/{flow_id}/{session_id}: OpenAI 실시간 음성에 연결하고, OpenAI 실시간 모델에 의해 도구로 플로우를 호출합니다. 이 접근 방식은 지연 시간이 짧은 애플리케이션에 이상적이지만, OpenAI 음성-음성 모델이 플로우를 호출하는 시기를 결정하기 때문에 덜 결정적(예측 가능성이 낮음)입니다.
  • /ws/flow_tts/{flow_id} 또는 /ws/flow_tts/{flow_id}/{session_id}: OpenAI 실시간 음성 전사(음성을 텍스트로 변환)를 사용하여 오디오를 텍스트로 변환한 다음, 각 전사본에 대해 직접 각 플로우가 호출됩니다. 이 접근 방식은 더 결정적(예측 가능성이 높음)이지만 지연 시간이 더 깁니다. 이는 Langflow Playground에서 사용되는 모드입니다.

경로 매개변수:

  • flow_id: 필수 경로 매개변수. 도구로 사용될 플로우의 ID입니다.
  • session_id: 선택적 경로 매개변수. 대화 세션을 위한 고유 식별자입니다. 제공하지 않으면 자동으로 생성됩니다.
