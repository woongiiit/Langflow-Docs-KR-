Models

모델 (Model) 컴포넌트

: Components > Models

모델 컴포넌트는 대형 언어 모델(LLM)을 사용하여 텍스트를 생성합니다.

파라미터에 대한 자세한 사항은 각 컴포넌트의 문서를 참고하세요.

플로우에서 모델 컴포넌트 사용하기

모델 컴포넌트는 텍스트 생성을 위한 입력(input)과 프롬프트(prompt)를 받아, 생성된 텍스트를 출력 컴포넌트로 전송합니다.

모델 출력은 Language Model 포트로 전송된 후 Parse Data 컴포넌트로 전달되어 구조화된 데이터 객체로 파싱될 수도 있습니다.

아래 예시는 챗봇을 OpenAI 모델을 사용해 만든 플로우입니다. 자세한 내용은 Basic prompting flow를 참조하세요.

Groq

이 컴포넌트는 Groq의 언어 모델을 사용하여 텍스트를 생성합니다.

  1. 플로우에서 이 컴포넌트를 사용하려면, Basic prompting flow와 같이 Model로 연결하거나 Agent 컴포넌트를 사용하는 경우 Model Provider로 선택하세요.
  1. Groq API Key 필드에 본인의 Groq API 키를 붙여넣습니다. Groq 모델 컴포넌트는 최신 모델 목록을 자동으로 가져옵니다. 목록을 새로고침하려면 Refresh 버튼을 클릭하세요.
  1. Model 필드에서 사용하려는 Groq 모델을 선택합니다. 이 예시에서는 실시간 대화 인터페이스에 권장되는 llama-3.1-8b-instant를 사용합니다.
  1. Prompt 컴포넌트에 다음 텍스트를 입력하세요:
You are a helpful assistant who supports their claims with sources.
  1. Playground를 열고 Groq LLM에 질문을 입력하면, 응답에 소스 목록이 포함됩니다.

자세한 내용은 Groq 문서를 참조하세요.

Hugging Face API

이 컴포넌트는 Model ID 필드에 지정된 모델을 사용하여 Hugging Face API에 요청을 보내고 텍스트를 생성합니다.

Hugging Face API는 Hugging Face Hub에 호스팅된 모델을 위한 호스팅 추론 API로, 인증을 위해 Hugging Face API 토큰이 필요합니다.

아래 예시는 Basic prompting flow를 기반으로 한 예시로, Hugging Face API 모델 컴포넌트가 OpenAI 모델 컴포넌트를 대체합니다. 서로 다른 호스팅 모델을 선택해 보면, 결과가 어떻게 다른지 확인할 수 있습니다.

  1. Basic prompting flow를 만듭니다.
  1. OpenAI 모델 컴포넌트를 Hugging Face API 모델 컴포넌트로 교체합니다.
  1. Hugging Face API 컴포넌트의 API Token 필드에 자신의 Hugging Face API 토큰을 입력합니다.
  1. Playground를 열고 모델에 질문을 입력하여 응답을 확인합니다.
  1. 다양한 모델을 시도하여 성능 차이를 확인해 보세요.

자세한 내용은 Hugging Face 문서를 참조하세요.

파라미터

입력값

이름타입설명
model_idStringHugging Face Hub의 모델 ID를 지정합니다(예: "gpt2", "facebook/bart-large").
huggingfacehub_api_tokenSecretStringHugging Face API 토큰입니다.
temperatureFloat출력의 무작위성을 제어합니다. 범위: [0.0, 1.0]. 기본값: 0.7
max_new_tokensInteger생성할 최대 토큰 수를 지정합니다. 기본값: 512
top_pFloatNucleus sampling 파라미터입니다. 범위: [0.0, 1.0]. 기본값: 0.95
top_kIntegerTop-k 샘플링 파라미터입니다. 기본값: 50
model_kwargsDictionary모델에 전달할 추가 키워드 인수입니다.

출력값

이름타입설명
modelLanguageModel지정된 파라미터로 구성된 HuggingFaceHub 인스턴스입니다.

  1. Ollama 컴포넌트를 플로우에 연결합니다(예: Agent 컴포넌트의 Custom Model로).

Base URLStringOllama API 엔드포인트입니다. 기본값: http://localhost:11434

TemperatureFloat응답의 창의성(무작위성)을 제어합니다.
