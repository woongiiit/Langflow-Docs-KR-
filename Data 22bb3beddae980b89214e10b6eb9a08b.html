데이터 (Data) 컴포넌트

: Components > Data

데이터 컴포넌트(Data component)는 외부 데이터 소스에서 데이터를 불러와 플로우에 입력하는 역할을 합니다.

일부 데이터 컴포넌트는 데이터를 불러오는 과정에서

HTML 등 원시(raw) 데이터를 텍스트로 변환하거나,

불러온 파일의 타입을 검증하는 등 기본적인 전처리·타입 체크를 수행합니다.

1. 플로우에서 데이터 컴포넌트 사용하기

URL 데이터 컴포넌트는 여러 개의 URL로부터 콘텐츠를 불러올 수 있습니다.

  • 컴포넌트의 URLs 입력란에 불러오고 싶은 URL을 입력하세요.
  • 여러 개의 URL을 추가하려면 Add URL 버튼을 클릭하세요.
  • 또는, Message 타입 출력을 제공하는 다른 컴포넌트(예: Chat Input 컴포넌트)와 연결하여

    해당 컴포넌트로부터 URL 목록을 받아올 수도 있습니다.

아래 예(문서 수집 파이프라인)에서는 URL 컴포넌트가 원시 HTML 데이터를 불러와

텍스트 분할기(Text Splitter)로 전달하고,

이후 벡터 데이터베이스가 이를 임베딩 및 인덱싱합니다.

2. 주요 데이터 컴포넌트별 설명

API 요청 (API Request)

  • 이 컴포넌트는 URL 또는 cURL 명령어를 사용해 HTTP 요청을 보냅니다.
  • Data 출력을, 입력을 받을 수 있는 다른 컴포넌트(예: Chat Output)와 연결해 사용할 수 있습니다.

사용 방법

  • URLs 입력란에 엔드포인트를 입력합니다.

    예: https://dummy-json.mock.beeceptor.com/posts (블로그 게시글 API)

  • Method(요청 방식)에는 GET, POST, PATCH, PUT, DELETE 중 하나를 입력할 수 있습니다.
  • Use cURL 버튼을 켜면 cURL 명령어를 붙여넣는 입력란이 추가됩니다.

    (예시: curl -v https://dummy-json.mock.beeceptor.com/posts)

  • Playground에서 Run Flow를 클릭하면 요청 결과가 Result에 표시됩니다.

디렉토리(Directory)

  • 지정한 디렉토리에서 파일을 재귀적으로 불러옵니다.
  • 파일 타입, 탐색 깊이, 동시처리(concurrency) 등 다양한 옵션 지원

파일(File)

  • 다양한 형식의 파일을 불러와 파싱 후 Data 객체로 변환
  • 멀티파일, 병렬처리, 에러 무시 등 옵션 지원

사용 방법

  1. Select files 버튼 클릭
  1. 로컬 파일 선택 또는 File management로 불러온 파일 선택 후 Select file 클릭
  1. 불러온 파일명이 컴포넌트에 표시됨
  • 기본 최대 지원 파일 크기는 100MB
  • 변경하려면 -max-file-size-upload 옵션 참고

입력 파라미터

이름표기설명

출력

messageMessageMessage 객체로 반환된 파일 내용

지원 파일 유형

  • 텍스트(.txt), 마크다운(.md, .mdx), CSV(.csv), JSON(.json), YAML(.yaml, .yml), XML(.xml), HTML(.html, .htm)
  • PDF(.pdf), Word(.docx), Python(.py), Shell(.sh), SQL(.sql), JavaScript(.js), TypeScript(.ts, .tsx)
  • 아카이브(.zip, .tar, .tgz, .bz2, .gz)

SQL 쿼리(SQL Query)

  • 지정한 데이터베이스에 SQL 쿼리를 실행하는 컴포넌트

입력 파라미터

출력

이름표기설명
resultResultSQL 쿼리 실행 결과

URL

  • 여러 URL에서 콘텐츠를 불러오고,

    텍스트나 원시 HTML 등 다양한 형식으로 반환할 수 있는 컴포넌트

사용 방법

  • URLs 입력란에 불러올 URL 입력(여러 개 추가 가능)
  • Max Depth 옵션으로 크롤링(탐색) 깊이 지정(예: 1이면 해당 페이지만, 2면 한 단계 링크까지)
  • 연결된 컴포넌트에 따라 DataFrame 또는 Message(텍스트) 등 원하는 형식으로 결과 전달
  • 크롤링 범위를 같은 도메인으로 제한하거나, 비동기 로딩(use_async) 등 고급 옵션 사용 가능

출력 연결 예시

  • URL → DataFrame → Chat Output : 표 형태 결과를 채팅창에 출력
  • URL → Message → Regex Extractor : HTML 원문에서 정규표현식으로 원하는 정보 추출

prevent_outsidePrevent Outside같은 도메인만 크롤링

dataframeDataFrame표 형태로 변환된 콘텐츠

웹훅(Webhook)

  • 외부 HTTP POST 요청이 들어오면 플로우를 자동으로 실행하는 트리거 컴포넌트
  • 입력이 유효한 JSON이 아니면 payload 객체로 감싸 처리함
  • Webhook 컴포넌트를 플로우에 추가하면 API 액세스 패널에 Webhook cURL 코드가 노출됨

활용 예시

  • Webhook → Parser → Chat Output 으로 연결하면

    외부에서 POST로 데이터를 보내 플로우를 즉시 실행하고,

    결과를 채팅창에 바로 확인 가능

입력 파라미터

이름표기설명
dataPayloadHTTP POST로 외부 시스템에서 받은 데이터
curlcURL해당 Webhook 요청용 cURL 명령 템플릿
endpointEndpointWebhook가 요청을 받는 URL 엔드포인트

출력

이름표기설명
output_dataData처리된 데이터. 입력이 없으면 빈 Data 반환, 유효 JSON 아닐 경우 payload로 감싸 반환
