임베디드 채팅 위젯

Publish 패널의 Embed into site 탭에는 HTML의 <body> 태그 안에 삽입하여 플로우와 상호작용할 수 있는 코드가 표시됩니다.

채팅 위젯은 langflow-chat이라는 웹 컴포넌트(web component: 재사용 가능한 사용자 정의 HTML 요소)로 구현되며 CDN(Content Delivery Network: 콘텐츠 전송 네트워크)에서 로드됩니다. 자세한 내용은 langflow-embedded-chat 저장소를 참조하세요.

샌드박스 예제는 Langflow 임베디드 채팅 CodeSandbox에서 확인할 수 있습니다.

다음 예제에는 HTML 코드에서 채팅 위젯을 사용하기 위한 최소 필수 입력값(React에서는 props라고 함)인 host_url과 flow_id가 포함되어 있습니다. host_url 값은 반드시 HTTPS여야 하며, URL 뒤에 /를 포함해서는 안 됩니다. flow_id 값은 Langflow URL에서 찾을 수 있습니다.

https://c822-73-64-93-151.ngrok-free.app/flow/dcbed533-859f-4b99-b1f5-16fce884f28f에서 기본 프롬프팅 플로우를 실행하는 Langflow 서버의 경우, 채팅 위젯 코드는 다음과 유사합니다:

<html>
  <head>
    <script src="https://cdn.jsdelivr.net/gh/logspace-ai/langflow-embedded-chat@main/dist/build/static/js/bundle.min.js"></script>
  </head>
  <body>
    </langflow-chat>
  </body>
</html>

이 코드가 HTML 내에 임베드되면 기본 프롬프팅 플로우에 의해 구동되는 반응형 챗봇이 됩니다.

  • 채팅 위젯을 추가로 구성하려면 추가 props를 포함하세요.
  • 모든 props와 해당 유형은 index.tsx에 나열되어 있습니다.
  • 채팅 위젯에 스타일을 추가하려면 JSON으로 요소를 사용자 지정하세요:
</langflow-chat>

Langflow 서버에 인증을 위한 API 키와 사용자 정의 세션 ID 값을 추가하려면:

<html>
  <head>
    <script src="https://cdn.jsdelivr.net/gh/logspace-ai/langflow-embedded-chat@main/dist/build/static/js/bundle.min.js"></script>
  </head>
  <body>
    </langflow-chat>
  </body>
</html>

채팅 위젯이 통신하려면 플로우에 Chat Input(채팅 입력)과 Chat Output(채팅 출력) 컴포넌트가 포함되어야 합니다. Chat Input 없이 Langflow에 메시지를 보내도 플로우는 트리거되지만, LLM은 메시지가 비어 있다고 경고합니다. Text Input(텍스트 입력)과 Text Output(텍스트 출력) 컴포넌트는 Langflow와 메시지를 주고받을 수 있지만, 지속적인 LLM "채팅" 컨텍스트 없이 작동합니다.

React로 채팅 위젯 임베드하기

React 애플리케이션에서 채팅 위젯을 사용하려면 위젯 스크립트를 로드하고 채팅 인터페이스를 렌더링하는 컴포넌트를 만드세요:

  1. 웹 컴포넌트를 선언하고 React 컴포넌트로 캡슐화합니다.
declare global {
  namespace JSX {
    interface IntrinsicElements {
      "langflow-chat": any;
    }
  }
}

export default function ChatWidget({ className }) {
  return (
    <div className={className}>
      </langflow-chat>
    </div>
  );
}
  1. 컴포넌트를 코드 어디에나 배치하여 채팅 위젯을 표시합니다.

예를 들어, 이 문서 세트에서 React 위젯 컴포넌트는 docs > src > components > ChatWidget > index.tsx에 위치합니다. index.tsx에는 CDN에서 채팅 위젯 코드를 로드하고 Langflow 서버를 가리키는 props로 ChatWidget 컴포넌트를 초기화하는 스크립트가 포함되어 있습니다.

import React, { useEffect } from 'react';

// 채팅 위젯 스크립트를 로드하는 컴포넌트
const ChatScriptLoader = () => {
  useEffect(() => {
    if (!document.querySelector('script[src*="langflow-embedded-chat"]')) {
      const script = document.createElement('script');
      script.src = 'https://cdn.jsdelivr.net/gh/langflow-ai/langflow-embedded-chat@main/dist/build/static/js/bundle.min.js';
      script.async = true;
      document.body.appendChild(script);
    }
  }, []);

  return null;
};

declare global {
  namespace JSX {
    interface IntrinsicElements {
      "langflow-chat": any;
    }
  }
}

export default function ChatWidget({ className }) {
  return (
    <div className={className}>
      <ChatScriptLoader />
      </langflow-chat>
    </div>
  );
}
  1. 페이지에 컴포넌트를 가져오려면 사이트에 다음을 추가하세요.
import ChatWidget from '@site/src/components/ChatWidget';
  1. 위젯을 페이지에 추가하려면 <ChatWidget className="my-chat-widget" />를 포함하세요.

Angular로 채팅 위젯 임베드하기

Angular 애플리케이션에서 채팅 위젯을 사용하려면 위젯 스크립트를 로드하고 채팅 인터페이스를 렌더링하는 컴포넌트를 만드세요.

Angular에서는 langflow-chat과 같은 커스텀 웹 컴포넌트를 컴포넌트에서 명시적으로 허용해야 하므로, Angular 템플릿에 <langflow-chat> 요소를 추가하고 Angular가 이를 인식하도록 구성해야 합니다. 이를 위해 모듈 구성에 CUSTOM_ELEMENTS_SCHEMA를 추가하세요.

모듈 구성에 CUSTOM_ELEMENTS_SCHEMA를 추가하려면 다음을 수행하세요:

  1. langflow-chat 웹 컴포넌트를 추가하려는 .module.ts 모듈 파일을 엽니다.
  1. .module.ts 파일 맨 위에 CUSTOM_ELEMENTS_SCHEMA를 가져옵니다:
import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
  1. @NgModule 데코레이터 내부의 schemas 배열에 CUSTOM_ELEMENTS_SCHEMA를 추가합니다:
import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule
  ],
  schemas: [CUSTOM_ELEMENTS_SCHEMA],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }
  1. 컴포넌트의 .component.ts 파일에 langflow-chat 요소를 포함하여 채팅 위젯을 컴포넌트의 템플릿에 추가합니다:

chat_window_style과 bot_message_style과 같은 JSON 객체를 받는 스타일 속성의 경우, Angular의 속성 바인딩 구문인 [propertyName]을 사용하여 JavaScript 객체로 전달합니다.

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  template: `
    <div class="container">
      <h1>Langflow Chat Test</h1>
      </langflow-chat>
    </div>
  `,
  styles: [`
    .container {
      padding: 20px;
      text-align: center;
    }
  `]
})
export class AppComponent {
  title = 'Langflow Chat Test';
}

채팅 위젯 구성

위젯 API를 사용하여 채팅 위젯을 사용자 지정하세요.

유형이 JSON인 props는 {"key":"value"} 형식의 문자열화된 JSON으로 전달해야 합니다.

모든 props와 해당 유형은 index.tsx에 나열되어 있습니다.

속성유형설명
flow_idString필수. 컴포넌트와 연결된 플로우의 식별자.
host_urlString필수. 채팅 컴포넌트와 통신하기 위한 호스트 URL.
api_keyStringLangflow에 전송할 X-API-Key 헤더.
additional_headersJSONLangflow 서버에 전송될 추가 헤더.
session_idString무작위 세션 ID를 대체할 사용자 정의 세션 ID.
heightNumber채팅 창의 높이(픽셀).
widthNumber채팅 창의 너비(픽셀).
chat_positionStringtop-right또는bottom-left와 같은 채팅 창 위치.
start_openBoolean채팅 창이 기본적으로 열려 있어야 하는지 여부.
chat_window_styleJSON전체 채팅 창 외관.
chat_trigger_styleJSON채팅 트리거 버튼 스타일링.
bot_message_styleJSON봇 메시지 형식.
user_message_styleJSON사용자 메시지 형식.
error_message_styleJSON오류 메시지 형식.
input_styleJSON채팅 입력 필드 스타일링.
input_container_styleJSON입력 컨테이너 스타일링.
send_button_styleJSON전송 버튼 스타일링.
send_icon_styleJSON전송 아이콘 스타일링.
window_titleString채팅 창 헤더에 표시되는 제목.
placeholderString채팅 입력 필드의 자리 표시자 텍스트.
placeholder_sendingString메시지 전송 중 자리 표시자 텍스트.
onlineBoolean채팅 컴포넌트가 온라인 상태인지 여부.
online_messageString채팅이 온라인 상태일 때 사용자 정의 메시지.
input_typeString채팅 메시지의 입력 유형.
output_typeString채팅 메시지의 출력 유형.
output_componentString여러 출력이 있을 때의 출력 ID.
chat_output_keyString여러 출력이 있을 때 표시할 출력.
tweaksJSON플로우에 대한 추가 사용자 정의 조정.
