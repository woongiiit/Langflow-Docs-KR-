파일 관리

Langflow의 파일 관리 시스템에서 파일을 업로드, 저장 및 관리하세요.

파일 관리 시스템에 파일을 업로드하면 파일이 중앙 위치에 보관되어 여러 플로우(flows)에서 수동으로 반복 업로드할 필요 없이 파일을 재사용할 수 있습니다.

파일 업로드하기

파일 관리 시스템은 /files URL에서 이용할 수 있습니다. 예를 들어, Langflow를 기본 주소인 http://localhost:7860에서 실행 중이라면, 파일 관리 시스템은 http://localhost:7860/files에 위치해 있습니다.

로컬 컴퓨터에서 파일을 업로드하려면:

  1. http://localhost:7860/files의 내 파일(My Files) 창에서 업로드(Upload)를 클릭합니다.
  1. 업로드할 파일을 선택합니다. 그러면 파일이 Langflow에 업로드됩니다.

한 번에 여러 파일을 업로드할 수 있습니다. 업로드된 파일은 서로 다른 프로젝트에 저장된 플로우에서도 사용할 수 있습니다.

내 파일(My Files)에 저장된 파일은 이름 변경, 다운로드, 복제 또는 삭제할 수 있습니다.

프로젝트에서 파일을 삭제하려면, 파일 아이콘 위에 마우스를 올려 클릭하여 선택한 다음 삭제(Delete)를 클릭합니다. 한 번에 여러 파일을 삭제할 수 있습니다.

프로젝트에서 파일을 다운로드하려면, 파일 아이콘 위에 마우스를 올려 클릭하여 선택한 다음 다운로드(Download)를 클릭합니다. 한 번에 여러 파일을 다운로드할 수 있지만, 이들은 함께 .ZIP 파일로 저장됩니다.

업로드된 파일을 플로우에서 사용하기

업로드한 파일을 플로우에서 사용하려면:

  1. 플로우에 파일(File) 컴포넌트를 포함시킵니다.
  1. 로드할 문서를 선택하려면, 파일(File) 컴포넌트에서 파일 선택(Select files) 버튼을 클릭합니다.
  1. 업로드할 파일을 선택한 다음 파일 선택(Select file)을 클릭합니다. 로드된 파일 이름이 컴포넌트에 나타납니다.

파일(File) 컴포넌트를 플로우에서 사용하는 예시를 보려면, 문서 QA 튜토리얼 프로젝트(Document QA tutorial project)를 참조하세요.

참고

일회성 업로드를 선호한다면, 파일(File) 컴포넌트는 여전히 로컬 컴퓨터에서 직접 일회성 업로드를 허용합니다.

지원되는 파일 유형

지원되는 최대 파일 크기는 100MB입니다.

텍스트 파일:

  • .txt - 텍스트 파일
  • .md, .mdx - 마크다운(Markdown) 파일
  • .csv - CSV 파일(쉼표로 구분된 값)
  • .json - JSON 파일(JavaScript 객체 표기법)
  • .yaml, .yml - YAML 파일(YAML 아인트 마크업 랭귀지)
  • .xml - XML 파일(확장 마크업 언어)
  • .html, .htm - HTML 파일(하이퍼텍스트 마크업 언어)
  • .pdf - PDF 파일(포터블 문서 형식)
  • .docx - Word 문서
  • .py - Python 파일
  • .sh - 쉘 스크립트(Shell scripts)
  • .sql - SQL 파일(구조화 질의 언어)
  • .js - JavaScript 파일
  • .ts, .tsx - TypeScript 파일

아카이브 형식(여러 파일 번들링용):

  • .zip - ZIP 아카이브
  • .tar - TAR 아카이브
  • .tgz - Gzip으로 압축된 TAR 아카이브
  • .bz2 - Bzip2로 압축된 파일
  • .gz - Gzip으로 압축된 파일
