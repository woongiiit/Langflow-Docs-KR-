Travel planning agent

여러 Tool-calling 에이전트를 활용한 에이전트 기반 애플리케이션을 위한 Travel Planning Agent 플로우를 구축하세요.

에이전트는 LLM을 "두뇌"로 사용하여 연결된 도구 중에서 적절한 도구를 선택해 작업을 수행합니다.

이 플로우에서는 여러 Tool-calling 에이전트가 OpenAI LLM을 활용해 여행 일정을 계획합니다. 각 에이전트는 System Prompt 필드에 정의된 서로 다른 역할을 담당합니다.

Chat input에서 사용자가 원하는 여행지를 입력하면, 해당 정보가 City Selection 에이전트로 전달됩니다. Local Expert 에이전트는 선택된 도시에 기반해 추가 정보를 제공합니다. Travel Concierge는 7일간의 여행 일정을 Markdown 형식으로 조합합니다.

모든 에이전트는 Search API와 URL Content Fetcher 컴포넌트에 접근할 수 있으며, 여행 경비 계산은 Travel Concierge만 Calculator를 사용할 수 있습니다.

사전 요구사항

  • 실행 중인 Langflow 인스턴스
  • OpenAI API 키
  • Search API 키

Travel Planning Agent 플로우 생성 방법

  1. Langflow를 실행하세요.
  1. New Flow를 클릭하세요.
  1. Travel Planning Agent 플로우를 선택하세요. 이렇게 하면 여러 Tool-calling 에이전트를 활용한 에이전트 기반 애플리케이션을 실행할 수 있는 기본 플로우가 생성됩니다

Travel Planning Agent 플로우는 다음과 같은 컴포넌트로 구성됩니다.

  • 여러 Tool calling agent 컴포넌트: 연결된 LLM을 사용해 사용자의 입력을 분석하고, 연결된 도구 중에서 적절한 도구를 선택해 작업을 수행합니다.
  • Calculator 컴포넌트: 기본 산술 연산을 수행합니다.
  • URL Content Fetcher 컴포넌트: 지정된 URL에서 콘텐츠를 수집합니다.
  • Chat Input 컴포넌트: 채팅에서 사용자의 입력을 받습니다.
  • Chat Output 컴포넌트: 플로우의 출력을 채팅에 표시합니다.
  • OpenAI model 컴포넌트: 사용자 입력과 프롬프트를 OpenAI API로 전송하고, 응답을 받아옵니다.

Travel Planning Agent 플로우 실행 방법

  1. OpenAI 및 Search API 컴포넌트에 자격 증명을 추가하세요.
  1. Playground를 클릭해 채팅 세션을 시작하세요. Chat input 컴포넌트에 정의된 여행에 대해 상세하고 유용한 답변을 받을 수 있습니다.

이제 Chat input에서 Chat output까지 쿼리가 정상적으로 처리되었다면, Travel Planning Agent 플로우를 완료한 것입니다.
