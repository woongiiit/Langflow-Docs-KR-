로직 (Logic) 컴포넌트

: Components > Logic

Langflow의 Logic 컴포넌트는 라우팅, 조건 처리, 플로우 관리 기능을 제공합니다.

Logic 컴포넌트 사용법

이 플로우에서는 Loop 컴포넌트를 활용해 문서 요약을 위한 “for each” 스타일의 반복 처리를 구현합니다.

  1. File 컴포넌트가 로컬 머신의 텍스트 파일을 로드합니다.
  1. Parse Data 컴포넌트는 파일 내용을 구조화된 Data 객체 리스트로 변환합니다.
  1. Loop 컴포넌트는 이 Data 리스트를 하나씩 꺼내어 Prompt 컴포넌트로 전달하며 요약 작업을 수행합니다.
  1. 더 이상 처리할 데이터가 없으면 Done 루프가 활성화되어 전체 페이지 수를 집계하고, 추가 Prompt를 통해 문서 전체의 톤을 요약합니다.

이처럼 Parse Data의 Data List 출력과 Loop의 Data 입력을 연결하면, Langflow 내에서 구조화된 “for each” 요약 플로우를 간단히 표현할 수 있습니다.

조건 라우터 (If-Else 컴포넌트)

이 컴포넌트는 두 문자열을 비교해 메시지를 분기합니다. 지정된 연산자를 사용하여 두 텍스트를 평가한 뒤, 조건이 참일 경우 true_result 포트로, 거짓일 경우 false_result 포트로 메시지를 전달합니다.

연산자는 기본적으로 단일 문자열 비교를 수행하지만, 정규 표현식(regex) 매칭을 통해 여러 단어를 한꺼번에 검색할 수도 있습니다.

사용법: 정규표현식 매칭

  1. If-Else 컴포넌트의 Text Input 포트를 Chat Input 컴포넌트에 연결합니다.
  1. If-Else 컴포넌트 설정:
    • Match Text: .*(urgent|warning|caution).* (이 값 중 하나를 검색)
    • Operator: regex (정규표현식 매칭, 대소문자 구분)
    • Message: New Message Detected (옵션 필드, True와 False 포트로 메시지 전달)
  1. 두 개의 동일한 플로우를 만들어 메시지를 처리합니다. 각 흐름에 OpenAI, Prompt, Chat Output 컴포넌트를 연결합니다.
  1. 한 흐름은 If-Else 컴포넌트의 True 포트에, 다른 흐름은 False 포트에 연결합니다.
  1. 두 OpenAI 컴포넌트에 API 키를 추가합니다.
  1. 각 Prompt 컴포넌트에 경로별 동작을 입력합니다.
    • 매칭 시: "새 메시지가 수신되어 긴급 대기열에 추가되었습니다."
    • 매칭되지 않을 시: "새 메시지가 수신되어 백로그에 추가되었습니다."
  1. Playground를 엽니다.
  1. 메시지를 전송하고 If-Else 컴포넌트 평가에 따라 라우팅 결과를 확인합니다.

User: A new user was created.

AI: A new message has been received and added to the backlog.

User: Sign-in warning: new user locked out.

AI: A new message has been received and added to the Urgent queue. Please review it at your earliest convenience.

매개변수

Inputs

이름타입설명
input_textString비교할 기본 텍스트 입력
match_textString비교 대상 텍스트
operatorDropdown비교 연산자 (equals, not equals, contains, starts with, ends with, regex 중 선택). 기본값: equals
case_sensitiveBooleantrue일 때 대소문자 구분. regex 연산자에는 적용되지 않음. 기본값: false
messageMessage참/거짓 루트로 전달할 메시지. 선택적 필드
max_iterationsInteger최대 반복 횟수. 기본값: 10
default_routeDropdown최대 반복 시 택할 경로 (true_result 또는 false_result). 기본값: false_result

Outputs

이름타입설명
true_resultMessage조건이 참일 때의 출력 메시지
false_resultMessage조건이 거짓일 때의 출력 메시지

연산자 동작

  • equals: 정확히 일치하는지 비교
  • not equals: 정확히 일치하지 않음을 비교
  • contains: input_text에 match_text가 포함되어 있는지 확인
  • starts with: input_text가 match_text로 시작하는지 확인
  • ends with: input_text가 match_text로 끝나는지 확인
  • regex: 정규표현식 매칭 수행 (항상 대소문자 구분, case_sensitive 설정 무시)

Loop 컴포넌트

팁: 다른 Loop 컴포넌트 예시는 Research Translation Loop 템플릿을 참조하세요.

이 컴포넌트는 Data 객체 리스트를 순회하며, 한 번에 하나의 항목을 출력하고 루프 입력에서 결과를 집계합니다.

아래 예시에서 Loop 컴포넌트는 Load CSV 컴포넌트의 Item 포트를 통해 CSV 파일의 각 행을 반복 처리합니다. 처리된 구조화 데이터를 Chroma DB에 적재하는 동작이 Done 포트에 연결되어 있습니다.

1. Loop 컴포넌트는 Load CSV 컴포넌트의 Data를 받아 Item 포트로 출력합니다.
2. 각 CSV 행은 Structured Output 컴포넌트로 변환되어 구조화된 데이터로 처리됩니다. 점선으로 표시된 Structured Output의 Looping 포트는 루프가 재시작되는 지점을 나타냅니다.
3. 더 이상 추출할 행이 없으면 Done 포트에 연결된 동작이 수행됩니다. 여기서는 데이터를 Chroma DB에 로드합니다.

Run flow 컴포넌트

이 컴포넌트는 Langflow 데이터베이스에 저장된 플로우를 에디터를 열지 않고 실행할 수 있도록 합니다.

Run Flow 컴포넌트는 에이전트(Agent)에 툴(tool)로 연결할 수도 있습니다. 에이전트가 사용할 도구의 이름과 설명 메타데이터는 자동으로 생성됩니다.

사용법

  1. Simple Agent 플로우에 Run Flow 컴포넌트를 추가합니다.
  1. Flow Name 메뉴에서 실행할 서브플로우를 선택합니다. 컴포넌트가 해당 플로우의 입력과 출력을 반영하도록 변경됩니다.
  1. Run Flow 컴포넌트에서 Tool Mode를 활성화합니다.
  1. Run Flow 컴포넌트를 에이전트의 Toolset 입력에 연결합니다.
  1. 플로우를 실행하면 에이전트가 해당 서브플로우를 도구로 사용합니다.

매개변수

Inputs

이름타입설명
flow_name_selectedDropdown실행할 플로우 이름
flow_tweak_dataDict플로우 동작을 맞춤 설정할 딕셔너리
dynamic inputsVarious선택된 플로우에 따라 생성된 추가 입력

Outputs

이름타입설명
run_outputsList[...]플로우 실행 결과로 생성된 모든 출력(Data, Message, DataFrame)

Legacy 컴포넌트

Legacy 컴포넌트는 더 이상 공식적으로 지원되지는 않지만, 여전히 사용은 가능합니다.

데이터 조건 라우터 (Data Conditional Router)

중요: 이 컴포넌트는 Langflow 버전 1.3부터 더 이상 활발히 개발되지 않습니다.

이 컴포넌트는 지정된 키(key)에 대해 조건을 평가해 Data 객체를 분기합니다. 단일 Data 객체 또는 Data 객체 리스트를 처리할 수 있으며, boolean 검증도 지원합니다.

Inputs

이름타입설명
data_inputData처리할 Data 객체 또는 Data 객체 리스트. 단일 항목과 리스트 모두 지원합니다.
key_nameString평가할 Data 객체의 키 이름
operatorDropdown적용할 연산자. 기본값은 equals. 사용 가능한 옵션:
equals, not equals, contains, starts with, ends with, boolean validator
compare_valueString비교할 값. boolean validator 연산자일 때는 사용되지 않습니다.

Outputs

이름타입설명
true_outputData/List조건을 만족한 객체들의 출력
false_outputData/List조건을 만족하지 않은 객체들의 출력

연산자 동작 방식

  • equals

    키의 값과 compare_value가 정확히 일치하는지 비교합니다.

  • not equals

    정확히 일치하지 않을 때 참입니다.

  • contains

    키의 값 안에 compare_value가 포함되어 있는지 확인합니다.

  • starts with

    키의 값이 compare_value로 시작하는지 확인합니다.

  • ends with

    키의 값이 compare_value로 끝나는지 확인합니다.

  • boolean validator

    키의 값을 불리언처럼 해석합니다. 다음 값들은 참으로 간주됩니다:

    • Boolean true
    • 문자열 "true", "1", "yes", "y", "on" (대소문자 무시)

      그 외의 값은 Python의 bool() 함수로 변환합니다.

리스트 처리

Data 객체 리스트를 입력받았을 때 다음과 같이 동작합니다:

  1. 리스트의 각 객체를 개별 평가
  1. 조건을 만족하는 객체는 true_output
  1. 조건을 만족하지 않는 객체는 false_output
  1. 모든 객체가 한쪽으로만 라우팅되면, 다른 출력은 빈 리스트가 됩니다.

중단된 컴포넌트 (Deprecated components)

중단된 컴포넌트는 더 이상 개발되지 않으며, 새로운 프로젝트에서는 사용을 권장하지 않습니다.

Flow as tool (Flow as tool 컴포넌트)

중요: 이 컴포넌트는 Langflow 버전 1.1.2부터 지원이 중단되었습니다. 대신 Run flow 컴포넌트를 사용하세요.

로드된 플로우를 실행하는 함수를 기반으로 도구(tool)를 생성합니다.

Inputs

이름타입설명
flow_nameDropdown실행할 플로우의 이름
tool_nameString생성할 도구의 이름
tool_descriptionString도구에 대한 설명
return_directBooleantrue로 설정하면, 도구 결과를 바로 반환

Outputs

이름타입설명
api_build_toolTool생성된 도구 객체

Sub flow (SubFlowComponent)

중요: 이 컴포넌트는 Langflow 버전 1.1.2부터 지원이 중단되었습니다. 대신 Run flow 컴포넌트를 사용하세요.

Inputs

이름타입설명

Outputs

이름타입설명
