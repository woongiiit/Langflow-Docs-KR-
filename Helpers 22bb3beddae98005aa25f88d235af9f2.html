Helper 컴포넌트

: Components > Helpers

Langflow의 Helper 컴포넌트는 플로우(flow) 안에서 데이터를 관리하거나 작업을 수행하고, 다른 컴포넌트와의 상호작용을 돕는 유틸리티 기능을 제공합니다.

플로우에서 Helper 컴포넌트 사용하기

Langflow에서의 채팅 메모리(chat memory)는 두 가지 방식으로 저장할 수 있습니다.

  1. LCBufferMemory를 이용한 Langflow 내부 테이블 저장
  1. 외부 데이터베이스에 연결하여 저장

Store Message Helper 컴포넌트는 채팅 내용을 Data 객체로 저장하며, Message History 컴포넌트는 저장된 채팅 내용을 데이터 객체 또는 문자열로 불러옵니다.

Batch Run

Batch Run 컴포넌트는 DataFrame의 텍스트 컬럼을 대상으로 **언어 모델(LLM)**을 실행하고, 원본 텍스트와 모델 응답을 포함한 새로운 DataFrame을 반환합니다.

출력 컬럼 구성

  • text_input: 입력 DataFrame의 원본 텍스트
  • model_response: 각 입력에 대한 모델의 응답
  • batch_index: 처리 순서를 나타내는 0부터 시작하는 인덱스
  • metadata (선택): 처리 관련 추가 정보

이 컬럼들은 Parser 컴포넌트와 연결되었을 때, 중괄호({}) 안에서 변수처럼 사용할 수 있습니다.

Batch Run + Parser 컴포넌트 사용법

  1. 언어 모델 컴포넌트를 Batch Run의 Language model 포트에 연결합니다.
  1. CSV 파일 등 DataFrame을 출력하는 컴포넌트를 Batch Run의 DataFrame 입력에 연결합니다.
  1. Batch Run의 Batch Results 출력을 Parser 컴포넌트의 입력으로 연결합니다.

Batch Run 구성 예시:

  • Column Name: 불러온 데이터에서 처리할 컬럼 이름 입력 (예: name)
  • System Message (선택): LLM에게 모든 행에 대해 어떻게 응답할지 안내하는 메시지 입력 (예: 각 이름에 대한 명함을 만들어줘)

Parser 구성 예시:

  • Template 필드: Batch Run에서 생성된 컬럼을 다음과 같이 사용 가능:
record_number: {batch_index}, name: {text_input}, summary: {model_response}

실행 방법:

Parser 컴포넌트에서 Run component를 클릭하여 실행하고, View table 버튼으로 결과를 확인합니다.

필요 시 Chat Output 컴포넌트를 연결하거나 Playground에서 결과를 시각적으로 확인할 수 있습니다.

Message History

💡 Langflow 1.1 이전에는 "Chat Memory" 컴포넌트로 불렸습니다.

Message History 컴포넌트는 Langflow 내부 또는 외부 메모리에서 채팅 메시지를 불러오는 기능을 합니다.

예시: Message Store 컴포넌트로 전체 대화를 저장하면, Message History가 이를 불러와 LLM의 응답에 활용할 수 있는 컨텍스트로 제공합니다.

입력값

이름타입설명
memoryMemory외부 메모리 사용 시 입력 (비워두면 Langflow 기본 테이블 사용)
senderString특정 발신자 유형으로 필터링
sender_nameString발신자 이름으로 필터링
n_messagesInteger불러올 메시지 수
session_idString채팅 세션 ID (없으면 현재 세션 사용)
orderString메시지 정렬 순서
templateString메시지 데이터 포맷 (예: {text}, {sender} 등 사용 가능)

출력값

이름타입설명
messagesData메시지를 데이터 객체로 반환
messages_textMessage텍스트 형태로 반환
dataframeDataFrameDataFrame 형태로 반환

Message Store

Message Store는 채팅 메시지나 텍스트를 Langflow 테이블 또는 외부 메모리에 저장하는 컴포넌트입니다.

예시: 전체 채팅 내용을 로컬 테이블에 저장하고, 이후 Message History로 불러와 LLM의 응답 맥락으로 활용합니다.

입력값

이름타입설명
messageString저장할 메시지 내용 (필수)
memoryMemory외부 메모리 사용 시 입력
senderString발신자 유형 (Machine 또는 User)
sender_nameString발신자 이름 (예: AI, User)
session_idString채팅 세션 ID (없으면 현재 세션 사용)

출력값

이름타입설명
stored_messagesList[Data]저장된 메시지 목록

Structured Output

이 컴포넌트는 LLM의 응답을 구조화된 데이터 형식으로 변환합니다.

예시: Financial Report Parser 템플릿에서는 구조화되지 않은 재무 보고서를 이 컴포넌트를 통해 정형 데이터로 변환합니다.

사용 방법

  1. Format Instructions 필드를 통해 LLM에 출력 구조 지침을 제공합니다. 이 값은 system prompt로 사용됩니다.
  1. Output Schema 테이블을 통해 출력 데이터의 구조를 정의합니다.

Output Schema 구성

항목설명
Name출력 필드 이름
Description필드의 용도 설명
Type데이터 타입 (str, int, float, bool, list, dict)
Multiple여러 값을 받을 경우 True (기본값: True)

예시 Template

EBITDA: {EBITDA}, Net Income: {NET_INCOME}, GROSS_PROFIT: {GROSS_PROFIT}

Playground에서는 다음과 같은 형태로 출력됩니다:

EBITDA: 900 million, Net Income: 500 million, GROSS_PROFIT: 1.2 billion

Output Parser

언어 모델의 출력값을 특정 형식으로 변환하는 파서입니다.

예를 들어, CSV 형식 파싱을 지원하여 응답을 쉼표로 구분된 리스트로 변환할 수 있습니다.

⚠️ 이 컴포넌트는 포맷 지침과 파싱 기능만 제공합니다. 실제 프롬프트는 Prompt 컴포넌트를 통해 작성해야 합니다.

사용 시나리오

  • 간단한 응답 목록이 필요한 경우 (["item1", "item2", "item3"])
  • 복잡한 필드/타입이 필요한 경우에는 Structured Output 사용 권장

사용 방법

  1. Prompt 컴포넌트를 생성하고 format_instructions 출력을 연결
  1. Prompt 템플릿에서 {format_instructions} 포함:
{format_instructions}
Please list three fruits.
  1. output_parser 출력을 LLM에 연결하면, 응답이 Python 리스트 형태로 변환됩니다:
["apple", "banana", "orange"]

이름설명

출력설명
format_instructions프롬프트에 포함시킬 포맷 지침
