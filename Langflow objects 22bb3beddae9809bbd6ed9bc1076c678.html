Langflow 객체

Langflow에서 객체는 데이터의 구조화된 기능적 표현 역할을 하는 Pydantic 모델입니다. Pydantic은 데이터 검증과 설정 관리를 위한 Python 라이브러리로, 타입 힌트를 사용하여 데이터를 검증합니다.

Data 객체

Data 객체는 데이터를 저장하고 조작하기 위한 컨테이너 역할을 하는 Pydantic 모델입니다.

주요 속성:

  • text_key: 주요 텍스트 데이터를 검색하기 위한 키를 지정합니다.
  • data: 추가 데이터를 저장하는 사전(dictionary)입니다.
  • default_value: text_key가 data 사전에 없을 때 사용되는 기본값입니다.

Data 객체 생성하기

키-값 쌍을 직접 할당하여 Data 객체를 생성할 수 있습니다. 예를 들면:

from langflow.schema import Data

# 지정된 키-값 쌍으로 Data 객체 생성하기
data = Data(text="my_string", bar=3, foo="another_string")

# 출력:
print(data.text)  # 출력: "my_string"
print(data.bar)   # 출력: 3
print(data.foo)   # 출력: "another_string"

text_key는 data 사전에서 주요 텍스트 내용으로 간주될 키를 지정합니다. default_value는 text_key가 없을 때 대체값을 제공합니다.

# 특정 text_key와 default_value로 Data 객체 생성하기
data = Data(data={"title": "Hello, World!"}, text_key="content", default_value="No content available")

# text_key와 default_value를 사용하여 주요 텍스트에 접근하기
print(data.get_text())  # 출력: "No content available" - "content" 키가 data 사전에 없기 때문

# 속성을 직접 호출하여 data 키에 접근하기
print(data.title)  # 출력: "Hello, World!" - "title" 키가 data 사전에 있기 때문

Data 객체는 출력 시각화에도 편리합니다. 출력 미리보기에는 데이터를 테이블로 검사하고 기본 유형의 셀을 팝업으로 볼 수 있는 시각적 요소가 있기 때문입니다. 이는 Langflow에서 복잡한 정보를 작업하고 시각화하는 통일된 방법을 만들기 위한 것입니다.

컴포넌트 입력에서 Data 객체를 수신하려면 DataInput 입력 유형을 사용하세요.

inputs = [
    DataInput(name="data", display_name="Data", info="수신되는 데이터 객체에 대한 유용한 정보", is_list=True),
]

Message 객체

Message 객체는 Data의 기능을 확장하여 채팅 상호작용을 위한 추가 속성과 메서드를 포함합니다.

핵심 메시지 데이터:

  • text: 메시지의 주요 텍스트 내용
  • sender: 발신자 식별자("User" 또는 "AI")
  • sender_name: 발신자 이름
  • session_id: 채팅 세션 식별자(string 또는 UUID - 범용 고유 식별자)
  • timestamp: 메시지가 생성된 시간(UTC)
  • flow_id: 흐름 식별자(string 또는 UUID)
  • id: 메시지의 고유 식별자

콘텐츠 및 파일:

  • files: 메시지와 관련된 파일 또는 이미지 목록
  • content_blocks: 구조화된 콘텐츠 블록 객체 목록
  • properties: 시각적 스타일 및 소스 정보를 포함한 추가 속성

메시지 상태:

  • error: 오류 발생 여부를 나타내는 불리언(Boolean) 값
  • edit: 메시지 편집 여부를 나타내는 불리언 값
  • category: 메시지 카테고리("message", "error", "warning", "info")

Message 객체는 Langflow 내에서 채팅 메시지를 보내고, 저장하고, 조작하는 데 사용할 수 있습니다.

Message 객체 생성하기

키-값 쌍을 직접 할당하여 Message 객체를 생성할 수 있습니다. 예를 들면:

from langflow.schema.message import Message

message = Message(text="Hello, AI!", sender="User", sender_name="John Doe")

컴포넌트 입력에서 Message 객체를 수신하려면 MessageInput 입력 유형을 사용하거나, Message 객체의 text 필드만 추출하려는 경우에는 MessageTextInput을 사용할 수 있습니다.

ContentBlock 객체

ContentBlock 객체는 여러 ContentTypes의 목록입니다. 이를 통해 이미지, 비디오, 텍스트 등 여러 유형의 콘텐츠를 단일 Message 내에 포함할 수 있습니다.

콘텐츠 유형은 content_types.py의 유형에서 구성된 Pydantic 기본 클래스입니다.

각 콘텐츠 유형에는 데이터 유형과 관련된 특정 필드가 있습니다. 예를 들면:

  • TextContent는 텍스트 문자열을 저장하기 위한 text 필드가 있습니다.
  • MediaContent는 미디어 파일 URL을 저장하기 위한 urls 필드가 있습니다.
  • CodeContent는 코드 스니펫을 위한 code와 language 필드가 있습니다.
  • JSONContent는 임의의 JSON 데이터를 저장하기 위한 data 필드가 있습니다.
  • ToolContent는 도구의 입력 매개변수를 저장하기 위한 tool_input 필드가 있습니다.

ContentBlock 객체 생성하기

다양한 콘텐츠 유형의 목록으로 ContentBlock 객체를 생성합니다.

content_block = ContentBlock(
    title="혼합 콘텐츠 예제",
    contents=[
        TextContent(text="이것은 텍스트 콘텐츠입니다"),
        MediaContent(urls=["http://example.com/image.jpg"]),
        JSONContent(data={"key": "value"}),
        CodeContent(code="print('Hello')", language="python")
    ],
    media_url=["http://example.com/additional_image.jpg"]
)

메시지에 ContentBlocks 객체 추가하기

이 예제에서는 텍스트와 미디어 ContentBlock이 메시지에 추가됩니다.

from langflow.schema.message import Message
from langflow.schema.content_block import ContentBlock
from langflow.schema.content_types import TextContent, MediaContent

message = Message(
    text="메인 메시지 텍스트",
    sender="User",
    sender_name="John Doe",
    content_blocks=[
        ContentBlock(
            title="텍스트 블록",
            contents=[
                TextContent(type="text", text="이것은 텍스트 콘텐츠입니다")
            ]
        ),
        ContentBlock(
            title="미디어 블록",
            contents=[
                MediaContent(type="media", urls=["http://example.com/image.jpg"])
            ]
        )
    ]
)

DataFrame 객체

DataFrame 클래스는 Pandas의 DataFrame 클래스를 확장한 것으로, Langflow의 Data 객체와 원활하게 작동하도록 특별히 설계되었습니다. 이 클래스는 DataFrame과 Data 객체 목록 간의 변환을 위한 메서드를 포함합니다.

DataFrame 객체는 Data 객체 목록, 사전, 기존 DataFrame을 포함한 다양한 입력 형식을 받아들입니다.

DataFrame 객체 생성하기

다양한 데이터 형식을 사용하여 DataFrame 객체를 생성할 수 있습니다:

from langflow.schema import Data
from langflow.schema.data import DataFrame

# Data 객체 목록에서 생성
data_list = [Data(data={"name": "John"}), Data(data={"name": "Jane"})]
df = DataFrame(data_list)

# 사전 목록에서 생성
dict_list = [{"name": "John"}, {"name": "Jane"}]
df = DataFrame(dict_list)

# 목록의 사전에서 생성
data_dict = {"name": ["John", "Jane"], "age": [30, 25]}
df = DataFrame(data_dict)

주요 메서드:

  • to_data_list(): DataFrame을 Data 객체 목록으로 다시 변환합니다.
  • add_row(data): DataFrame에 단일 행(Data 객체 또는 사전)을 추가합니다.
  • add_rows(data): DataFrame에 여러 행(Data 객체 또는 사전 목록)을 추가합니다.

사용 예제:

# DataFrame 생성하기
df = DataFrame([Data(data={"name": "John"}), Data(data={"name": "Jane"})])

# 새 행 추가하기
df = df.add_row({"name": "Alice"})

# Data 객체 목록으로 다시 변환하기
data_list = df.to_data_list()

# pandas 기능 사용하기
filtered_df = df[df["name"].str.startswith("J")]

컴포넌트 입력에서 DataFrame 객체를 사용하려면 DataFrameInput 입력 유형을 사용하세요.

DataFrameInput(
    name="dataframe_input", display_name="DataFrame 입력", info="DataFrame 객체를 위한 입력입니다.", tool_mode=True
),
