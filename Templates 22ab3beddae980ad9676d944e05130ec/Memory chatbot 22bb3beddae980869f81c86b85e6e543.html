Memory chatbot

이 플로우는 기본 프롬프트 플로우에 메시지 히스토리(Message history) 컴포넌트를 추가하여, 최대 100개의 이전 채팅 메시지를 저장하고 이를 현재 대화의 컨텍스트로 활용합니다.

사전 요구사항

  • 실행 중인 Langflow 인스턴스
  • OpenAI API 키

메모리 챗봇 생성하기

  1. Langflow 대시보드에서 New Flow를 클릭하세요.
  1. Memory Chatbot를 선택하세요.
  1. Memory Chatbot 플로우가 생성됩니다.

이 플로우는 Basic Prompting 플로우에 Message history 컴포넌트를 추가합니다. 이 컴포넌트는 이전 메시지들을 가져와 Prompt 컴포넌트에 전달하여 템플릿의 일부에 맥락으로 채웁니다.

템플릿을 확인하려면 Prompt 컴포넌트에서 Template 필드를 클릭하세요. Prompt는 OpenAI 모델 컴포넌트에 입력에 어떻게 응답할지 지시합니다.

You are a helpful assistant that answers questions.

Use markdown to format your answer, properly embedding images and urls.

History:

{memory}

프롬프트의 {memory} 코드는 memory라는 새로운 입력 포트를 컴포넌트에 생성합니다. Message history 컴포넌트는 이 포트에 연결되어 Playground에서 온 채팅 메시지를 저장하고, OpenAI 컴포넌트에 이전 채팅 메시지의 메모리를 제공합니다.

메모리 챗봇 플로우 실행하기

  1. Playground를 여세요.
  1. 여러 가지 질문을 입력하세요. 예를 들어, 다음과 같은 대화를 입력해 보세요.
Hi, my name is Luca.
Please tell me about PostgreSQL.
What is my name?
What is the second subject I asked you about?

챗봇은 당신의 이름과 이전 질문들을 기억합니다.

  1. Message Logs 창을 보려면 Options를 클릭한 다음 Message Logs를 클릭하세요. Message Logs 창에는 모든 이전 메시지가 session_id별로 정렬되어 표시됩니다.

Memory Chatbot에서 Session ID 사용하기

session_id는 Langflow에서 AI와 사용자 간의 대화 세션을 저장하는 고유 식별자입니다. 대화가 시작될 때 session_id가 생성되고, 해당 세션 동안 오고 가는 모든 메시지에 연결됩니다.

당신이 만든 Memory Chatbot 플로우에서는 Message history 컴포넌트가 Session ID를 통해 이전 상호작용을 참조합니다. Session ID 값을 변경하면 서로 다른 대화 기록으로 전환되는 것을 확인할 수 있습니다.

  1. Message history와 Chat Input 컴포넌트의 Session ID 필드에 MySessionID와 같은 값을 입력하세요.
  1. 이제 Playground에서 새 메시지를 보내면 Message Logs 창에 새로운 메모리가 생성됩니다.
  1. 대화가 서로 다른 메모리 세션에 저장되는 방식을 확인해보세요.
