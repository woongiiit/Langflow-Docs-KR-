Inputs and outputs

: Components > IO

Langflow의 입력/출력(IO) 컴포넌트는 플로우(flow)의 시작과 끝에서 데이터를 주고받는 역할을 합니다. 사용자의 입력을 받아 플로우를 시작하거나, 플로우의 최종 결과를 사용자에게 보여줄 때 사용됩니다.

Chat Input 컴포넌트

Chat Input 컴포넌트는 텍스트, 파일, 메타데이터 등 다양한 입력 유형을 받아 Message 객체를 생성하며, 입력된 텍스트와 발신자 정보, 세션 ID, 파일 첨부 정보를 포함한 Message 객체를 반환합니다. Playground에서 대화형 채팅 인터페이스를 제공합니다. 필요 시 Message Store에 저장할 수도 있습니다.

입력값 (Inputs)

이름표시명설명
input_valueText전달할 메시지 텍스트
should_store_messageStore Messages메시지를 기록에 저장할지 여부
senderSender Type발신자 유형
sender_nameSender Name발신자 이름
session_idSession ID채팅 세션 고유 ID (비워둘 경우 현재 세션 사용)
filesFiles메시지와 함께 전송할 파일들
background_colorBackground Color아이콘 배경색
chat_iconIcon메시지 아이콘
text_colorText Color발신자 이름 텍스트 색

출력값 (Outputs)

이름타입설명
messageMessage지정된 속성을 포함한 생성된 Message 객체

Message 생성 방식

ChatInput 클래스는 입력값에 따른 Message 타입의 객체를 생성하고 저장할 수 있는 비동기 메서드를 제공합니다. Message 객체는 Message.create() 팩토리 메서드를 통해 ChatInput 클래스의 message_response 메서드 안에 생성됩니다.

message = await Message.create(
    text=self.input_value,
    sender=self.sender,
    sender_name=self.sender_name,
    session_id=self.session_id,
    files=self.files,
    properties={
        "background_color": background_color,
        "text_color": text_color,
        "icon": icon,
    },
)

Text Input 컴포넌트

Text Input 컴포넌트는 텍스트 문자열 입력을 받아 Message 객체를 생성하고, 입력된 텍스트만을 포함한 Message 객체를 반환합니다. 출력은 Playground에 표시되지 않습니다.

입력값 (Inputs)

이름타입설명
input_valueText출력으로 전달할 텍스트/내용

출력값 (Outputs)

이름타입설명
textText생성된 텍스트 메시지

Chat Output 컴포넌트

Chat Output 컴포넌트는 입력된 텍스트, 발신자 정보, 세션 ID, 스타일 속성을 포함한 Message 객체를 생성합니다. 입력으로 Data, DataFrame, Message를 받을 수 있습니다.

입력값 (Inputs)

이름표시명설명
input_valueText출력할 메시지 텍스트
should_store_messageStore Messages메시지를 기록에 저장할지 여부
senderSender Type발신자 유형
sender_nameSender Name발신자 이름
session_idSession ID채팅 세션 고유 ID (비워둘 경우 현재 세션 사용)
data_templateData TemplateData를 Text로 변환할 템플릿. 비워두면 Data 내부 text 키 사용
background_colorBackground Color아이콘 배경색
chat_iconIcon메시지 아이콘
text_colorText Color발신자 이름 텍스트 색
clean_dataBasic Clean DataDataFrame 입력을 Text로 변환 시 빈 행, 빈 셀 줄, 다중 개행 제거 여부

출력값 (Outputs)

이름타입설명
messageMessage지정된 속성을 포함한 생성된 Message 객체

Text Output 컴포넌트

Text Output 컴포넌트는 하나의 텍스트 입력을 받아 Message 객체를 생성하고 해당 텍스트를 포함한 Message 객체를 반환합니다. 출력은 Playground에 표시되지 않습니다.

입력값 (Inputs)

이름표시명설명
input_valueText출력할 텍스트

출력값 (Outputs)

이름타입설명
textText생성된 텍스트 메시지

예제 플로우

Chat Input 컴포넌트를 OpenAI 모델 컴포넌트의 입력 포트에 연결하고, OpenAI 모델 컴포넌트의 Message 출력을 Chat Output 컴포넌트에 연결합니다. 그리고 OpenAI 모델 컴포넌트의 API Key 필드에 OpenAI API 키를 입력하면 완성입니다.

Playground에서 메시지를 입력하면 Chat Input이 모델에 전달하고, 모델의 응답이 Chat Output으로 표시됩니다.

Chat 컴포넌트 예제 플로우 단계별 안내

  1. 컴포넌트 연결: Chat Input과 Chat Output 컴포넌트를 Message 타입을 처리할 수 있는 컴포넌트들에 연결합니다. 본 예제에서는 Chat Input을 OpenAI 모델의 Input 포트에, OpenAI 모델의 Message 포트를 Chat Output에 연결합니다.
  1. API 키 설정: OpenAI 모델 컴포넌트의 OpenAI API Key 필드에 본인의 API 키를 입력합니다.
  1. Playground에서 메시지 전송: Playground를 열고 메시지를 입력하면, OpenAI 모델 컴포넌트가 응답합니다. 필요하다면 System Message를 설정하여 응답 방식을 제어할 수 있습니다.
  1. 로그 확인: Langflow UI에서 플로우 이름을 클릭한 후 Logs 버튼을 클릭하면 로그 창이 열립니다.
  1. 첫 번째 메시지 조회: Chat Input에서 전송된 메시지를 확인하려면 Outputs를 클릭합니다:
"messages": [
  {
    "message": "What's the recommended way to install Docker on Mac M1?",
    "sender": "User",
    "sender_name": "User",
    "session_id": "Session Apr 21, 17:37:04",
    "stream_url": null,
    "component_id": "ChatInput-4WKag",
    "files": [],
    "type": "text"
  }
]
  1. 두 번째 메시지 조회: OpenAI 모델의 응답이 Chat Output으로 전송된 결과를 확인하려면 Outputs를 클릭합니다:
"outputs": {
  "text_output": {
    "message": "To install Docker on a Mac with an M1 chip, you should use Docker Desktop for Mac, which is optimized for Apple Silicon. Here's a step-by-step guide to installing Docker on your M1 Mac:\n\n1. ...",
    "type": "text"
  }
}
팁: 각 컴포넌트의 출력을 개별적으로 확인하려면 Inspect output을 클릭하세요.

API를 통한 채팅 메시지 전송

Chat Input 컴포넌트는 Langflow API에 메시지를 전달하는 진입점 역할을 합니다. 동일한 예시 메시지를 프로그래밍 방식으로 Langflow 서버에 전송하려면 다음 단계를 따라주세요.

  1. 엔드포인트 확인

    Langflow 대시보드에서 Publish를 클릭한 뒤, API access를 클릭하여 엔드포인트 URL을 확인합니다.

  1. cURL 명령 복사

    cURL 탭에서 명령어를 복사한 뒤 터미널에 붙여넣습니다. 예시는 다음과 같습니다:

    curl --request POST \
  --url 'http://localhost:7860/api/v1/run/51eed711-4530-4fdc-9bce-5db4351cc73a?stream=false' \
  --header 'Content-Type: application/json' \
  --data '{
    "input_value": "What'\''s the recommended way to install Docker on Mac M1?",
    "output_type": "chat",
    "input_type": "chat"
  }'
  1. input_value 수정하기

    "input_value" 필드에 실제 질문을 입력합니다. 예:

    "input_value": "What's the recommended way to install Docker on Mac M1?"
    • output_type과 input_type 파라미터를 함께 전달합니다.
    • chat 타입은 Playground에 메시지를 표시하고 추가 설정 옵션을 제공합니다.
    • text 타입은 단순 문자열 응답을 반환하며 Playground에는 표시되지 않습니다.
  1. 커스텀 session_id 추가

    메시지의 데이터 객체에 "session_id"를 추가하면, 클라이언트와 Langflow 서버 간에 새로운 채팅 세션이 시작됩니다. 예:

    curl --request POST \
  --url 'http://localhost:7860/api/v1/run/51eed711-4530-4fdc-9bce-5db4351cc73a?stream=false' \
  --header 'Content-Type: application/json' \
  --data '{
    "input_value": "What's the recommended way to install Docker on Mac M1?",
    "session_id": "docker-question-on-m1",
    "output_type": "chat",
    "input_type": "chat"
  1. POST 요청 전송

    터미널에서 명령을 실행하면, Langflow 서버가 응답을 반환합니다.

  1. Playground 확인

    Playground에 접속하면 docker-question-on-m1이라는 이름의 새로운 채팅 세션이 나타납니다.

  1. Tweaks로 추가 파라미터 수정

    Chat Input/Chat Output 컴포넌트의 Tweaks를 통해 더 많은 파라미터를 조정할 수 있습니다.

    • Publish → API access → Tweaks를 클릭하세요.
  1. 예: 메시지 저장 비활성화

    Chat Input 컴포넌트에서 메시지 저장(should_store_message)을 비활성화하려면 다음과 같이 tweaks 필드를 추가합니다:

    curl --request POST \
  --url 'http://localhost:7860/api/v1/run/51eed711-4530-4fdc-9bce-5db4351cc73a?stream=false' \
  --header 'Content-Type: application/json' \
  --data '{
    "input_value": "Text to input to the flow",
    "output_type": "chat",
    "input_type": "chat",
    "tweaks": {
      "ChatInput-4WKag": {
        "should_store_message": false
      }
    }
  }'

    이후 Logs 패널에서 요청을 확인하면 should_store_message: false가 적용된 것을 볼 수 있습니다.

*이 문서는 https://docs.langflow.org/components-io 문서를 번역/편집한 것으로 생략되거나 추가된 내용이 있습니다. 자세한 사항은 원문을 참고하시기 바랍니다.
